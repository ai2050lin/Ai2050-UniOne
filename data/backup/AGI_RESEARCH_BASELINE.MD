# AGI 里程碑基准测试方案（Baseline）

## 目标
围绕“实现通用人工智能（AGI）”建立可执行、可量化、可淘汰的分阶段测试体系。该体系用于评估模型是否具备跨领域泛化、结构化关联、快速读写、规模化稳定和安全可控能力。

## 设计原则
1. 每个里程碑都包含四类门槛：能力测试、鲁棒测试、效率测试、安全测试。
2. 不使用单一总分判定 AGI，必须多维度同时达标。
3. 每个节点必须验证规模化趋势（小/中/大模型曲线一致）。
4. 每个节点设置硬阈值与淘汰条件，避免“理论正确但工程无效”。

## Milestone 0：基础设施就绪
### 关键节点
- 评测平台搭建完成。
- 数据治理与泄漏审计机制上线。
- 训练-评测-记录全链路可追溯。

### 必过测试
- 同一配置重复训练 3 次，核心指标方差可控。
- 评测集泄漏审计通过。
- 实验记录包含版本、参数、随机种子、硬件与日志。

### 达标水平
- 95% 以上实验可复现。
- 所有核心结果可追溯到具体代码与数据版本。

## Milestone 1：通用表征与特征提取
### 关键节点
- 在文本、视觉、语音等多模态上形成可迁移表征。

### 必过测试
- 跨任务迁移测试。
- 少样本泛化测试。
- OOD（分布外）泛化测试。

### 达标水平
- 跨域迁移性能达到同规模模型前列。
- OOD 性能相对 ID 性能下降幅度受控（建议不超过 20%-30%）。

## Milestone 2：关联网络与组合泛化
### 关键节点
- 特征可组合、可关联，不依赖模板记忆。

### 必过测试
- 组合泛化测试（训练组合与测试组合解耦）。
- 关系推理测试。
- 跨模态检索与对齐测试。

### 达标水平
- 在组合拆分任务中显著优于记忆型基线。
- 跨模态关联性能稳定提升。

## Milestone 3：快速读写与持续学习
### 关键节点
- 新知识可快速写入，旧知识不显著遗忘。

### 必过测试
- 增量学习测试。
- 长上下文记忆测试。
- 知识更新时延与回写正确率测试。
- 灾难遗忘率测试。

### 达标水平
- 新知识写入延迟低、读取准确。
- 旧任务性能回退小（建议 <5%-10%）。

## Milestone 4：统一机制跨能力验证
### 关键节点
- 统一底层机制覆盖视觉、语言、推理等能力域。
- 架构主干一致，仅参数或任务头变化。

### 必过测试
- 同骨干多任务训练。
- 跨任务迁移与共享参数效率对比。
- 模块共享率与性能折中分析。

### 达标水平
- 共享骨干模型的多任务平均性能接近专用模型。
- 参数效率、训练效率优于独立多模型方案。

## Milestone 5：规划、工具使用与闭环执行
### 关键节点
- 从“问答能力”升级到“任务完成能力”。

### 必过测试
- 多步规划任务。
- 工具调用任务（检索、代码、执行）。
- 长时任务完成率与自纠错能力测试。

### 达标水平
- 复杂任务端到端成功率稳定。
- 出错后可自修复，非盲目重试。

## Milestone 6：开放世界鲁棒性
### 关键节点
- 对分布漂移、噪声和攻击输入保持韧性。

### 必过测试
- 对抗输入与红队测试。
- 提示注入与越权攻击测试。
- 多模态噪声、缺失、冲突输入测试。

### 达标水平
- 关键能力在攻击与扰动下仍可用。
- 安全策略触发率与误杀率达到可接受平衡。

## Milestone 7：规模化定律与效率拐点
### 关键节点
- 大规模下三大核心能力持续有效。

### 必过测试
- 参数规模扩展实验。
- 数据规模扩展实验。
- 训练算力与推理成本曲线测试。

### 达标水平
- 扩展曲线持续向上且无明显崩塌区。
- 单位性能成本持续下降。

## Milestone 8：安全可控的准 AGI 验收
### 关键节点
- 高自主性下仍可对齐、可审计、可回滚。

### 必过测试
- 长时目标一致性测试。
- 权限边界与策略遵循测试。
- 人工接管与紧急停机测试。
- 决策可解释与审计追踪测试。

### 达标水平
- 高风险场景零严重违规。
- 关键链路具备可追踪证据。
- 接管与回滚机制可靠生效。

## AGI 候选通过线（建议）
1. 至少 5 类能力域同时达到接近专家或高熟练人类水平。
2. 在开放任务中具备稳定自规划、自纠错与任务闭环能力。
3. 在参数与数据规模扩展时，核心能力不退化。
4. 安全与对齐指标连续多个评测周期达标。

## 执行建议（工作循环）
1. Extract：从现有深度网络抽取结构信号。
2. Abstract：形成可检验数学假设。
3. Implement：实现最小可运行新模块。
4. Evaluate：按里程碑门槛评测。
5. Decide：保留/淘汰/合并假设。
6. Archive：沉淀正负结果与复现实验记录。

## 附：统一评分看板（建议）
- Core-Capability Score：能力主分（多任务平均）。
- Robustness Score：鲁棒性主分（攻击与噪声下性能保持）。
- Efficiency Score：训练/推理/存储效率主分。
- Safety Score：对齐、安全、可控性主分。
- Scaling Score：扩展斜率主分（小/中/大规模一致性）。

建议按季度更新门槛，按月评估趋势，按周执行实验循环。
