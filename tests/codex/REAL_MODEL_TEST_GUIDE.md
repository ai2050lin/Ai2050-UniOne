# 阶段6真实模型测试指南

## 概述

本指南说明如何在本地真实LLM模型上验证阶段6的统一参数级编码理论。

## 前提条件

1. **硬件要求**:
   - GPU: NVIDIA RTX 4090D (已检测到)
   - 显存: 至少16GB (Qwen3-4B约8GB, DeepSeek-7B约14GB)

2. **软件要求**:
   - Python 3.8+
   - PyTorch with CUDA support
   - transformers库

3. **模型要求**:
   - Qwen3-4B 或 DeepSeek-7B 模型文件

## 模型配置

### 步骤1: 确定模型路径

将您的模型放在以下任一路径：
- `~/models/` (用户目录下的models文件夹)
- `./models/` (当前工作目录下的models文件夹)

或者，修改测试脚本中的`MODEL_CONFIGS`变量，指定实际的模型路径。

### 步骤2: 验证模型文件

确保模型目录包含以下文件之一：
- `model-00001-of-000xx.bin` (分片权重)
- `model.safetensors` (SafeTensors格式)
- `pytorch_model.bin` (PyTorch格式)

## 测试内容

### 测试1: Family Patch机制
- **目的**: 验证同一家族的概念共享基向量
- **测试概念**:
  - 水果家族: 苹果、香蕉、橙子、葡萄
  - 动物家族: 狗、猫、兔子、鸟
- **预期结果**:
  - 共享基比例 > 0.5
  - 同家族相似度 > 0.5

### 测试2: 多维度编码
- **目的**: 验证同一内容在不同风格下有不同的编码
- **测试文本**:
  - 聊天风格、论文风格、技术风格、诗歌风格
- **预期结果**:
  - 编码多样性 > 0.05

### 测试3: 动态编码
- **目的**: 验证概念在不同上下文中的编码变化
- **测试文本**:
  - 逐步丰富对"苹果"的描述
- **预期结果**:
  - 编码连续性 > 0.5

## 故障排查

### 问题1: 模型加载失败
- **症状**: `RuntimeError: Error(s) in loading state_dict...`
- **原因**: 模型文件损坏或不完整
- **解决**: 重新下载模型或检查文件完整性

### 问题2: CUDA内存不足
- **症状**: `CUDA out of memory`
- **原因**: GPU显存不足
- **解决**:
  1. 使用较小的模型（如Qwen2.5-0.5B）
  2. 减少batch_size（脚本默认为1）
  3. 使用CPU模式（较慢）

### 问题3: 找不到模型
- **症状**: `[WARNING] 未在常见路径找到模型`
- **原因**: 模型不在预期位置
- **解决**:
  1. 将模型移动到 `~/models/` 或 `./models/`
  2. 编辑脚本中的 `MODEL_CONFIGS` 指定正确路径

## 测试结果解读

### 成功指标
- **共享基比例高**: 说明同家族概念共享基向量
- **Offset稀疏度高**: 说明概念特异性信息集中在少数维度
- **同家族相似度高**: 说明家族内部概念编码相似
- **编码多样性高**: 说明不同风格有不同编码
- **编码连续性高**: 说明上下文变化时编码平滑过渡

### 失败可能原因
1. **模型训练不足**: 模型未充分学习概念语义
2. **上下文不足**: 测试文本可能需要更多上下文
3. **阈值设置不合理**: 需要根据实际模型调整阈值
4. **模型架构差异**: 不同架构的编码机制可能不同
