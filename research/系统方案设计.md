## 零、核心思想与战略转变

### 0.1 项目本质

深度神经网络提取了大脑中的某种数学结构，产生了语言能力。语言能力既包括微观/中观/宏观多角度的知识体系，也包括语言本身的风格/逻辑/语法。这说明背后有某种数学结构——这个数学结构很可能超过了现在的数学体系，需要跳出预设理论，根据大量拼图进行分析和还原。

最终目标：

> 从黑箱 Transformer → 可解释语义结构 + 动态生成机制

建立：**神经网络内部科学（Neural Network Internal Science）**

### 0.2 259个Phase的核心教训

过去的259个Phase存在**三大根本问题**：

1. **没有系统性规划**：每次实验基于上一个"灵感"，导致30+拼图全是统计性质，功能机制几乎空白
2. **理论预设陷阱**：12条理论路线全部被推翻（ratio=0.22恒定量、频谱力学预测logit、平坦线性几何、ICSPB统一变量...），每次都是先预设理论再找证据
3. **统计量≠机制**：积累了大量cos/norm/probe accuracy等统计量，但解释力只有5-7%，无法从文本预测编码

### 0.3 战略转变：从"描述统计"到"逆向还原"

**旧路线（失败）**：观察统计现象 → 猜测理论 → 验证理论 → 被推翻
**新路线（当前）**：系统填充拼图 → 临界点自然涌现 → 从拼图还原数学结构

关键洞察：**不要猜理论，让数据说话。当拼图填充率>60%且3个统合格子被填充时，数学结构会自然涌现。**

## 四、核心未解问题与突破路径

### 问题1：深层为何能"锁定"浅层干预？★★★★★

**现象**：概念替换干预在浅层switch_rate=0.67-0.83，深层=0.00
**已知**：浅层确实编码概念信息（PCA5探针0.98），但深层对替换免疫
**未知**：是注意力机制在"纠错"？还是FFN在"锁定"？还是残差连接的累积效应？
**突破路径**：
- 逐层追踪替换后的残差流变化，找到"纠正"发生的确切层
- 在该层分别消融注意力和FFN，观察是否恢复浅层干预的效果
- 这是最关键的实验，因为如果找到"锁定机制"就等于找到语言生成的核心控制逻辑

### 问题2：中间层template热点的功能是什么？★★★★

**现象**：Qwen3=L18, GLM4=L20, DS7B=L14-21出现template主导的PC
**已知**：这些层template方差占比98.8%，可能与"句式处理"相关
**未知**：template热点是"语法加工厂"还是"信息中转站"？
**突破路径**：
- 消融template热点层的template方向，观察输出是否失去语法结构
- 跨句式追踪template方向的共享度
- 如果template热点是语法编码的核心，直接填充GR-1a/GR-2a

### 问题3：知识×逻辑的统一编码机制 ★★★★

**现象**："not"不否定属性方向，而是添加正交扰动（delta_attr_cos≈0）
**已知**：知识编码和逻辑操作在不同维度上
**未知**：是否存在跨能力的统一编码基底？还是完全独立？
**突破路径**：
- SAE特征在知识/逻辑任务中的复用率分析
- 如果复用率>30%，说明存在统一基底（直接填充UN-1）

### 问题4：逻辑推理的因果载体 ★★★

**现象**：逻辑推理恢复率<4%，但逻辑信号是属性信号的2.7倍
**已知**：逻辑信号强，但干预无效
**未知**：逻辑推理的计算在哪个组件中完成？注意力头？FFN？组合？
**突破路径**：
- 注意力头级别因果追踪（需修复当前的消融方法）
- 在注意力输出层（非层输出）做消融

### 问题5：从"统计描述"到"可预测理论" ★★★

**现象**：解释力只有5-7%
**已知**：大量统计量但无法从文本预测编码
**未知**：给定一个新句子，能否预测其残差流编码？
**突破路径**：先解决问题1-4，积累足够拼图后尝试构建预测方程

## 五、实验系统架构

### 5.1 三元框架

```
分析维度（What） × 分析操作（How） × 评估指标（Score）
```

并通过闭环实现自动进化：

```
假设 → 实验 → 结果 → 归纳 → 新假设
```

#### 分析维度（7层 → 对应PUZZLE框架7大类）

| 维度 | 对应PUZZLE | 当前填充率 |
|------|-----------|-----------|
| 词汇层(Lexical) | WE | 14% |
| 句法层(Syntax) | GR | 8% |
| 语义层(Semantics) | KN | 9% |
| 概念层(Conceptual) | KN-3 | 9% |
| 上下文层(Contextual) | SE | 11% |
| 推理层(Reasoning) | LG | 7% |
| 生成层(Generation) | MG | 7% |

#### 分析操作（7类 → 从观察到因果的递进）

| 操作 | 层级 | 当前能力 | 接口规范 |
|------|------|---------|---------|
| Probe(探针) | 观察 | ✅ 成熟 | `probe(hidden, label) → accuracy` |
| Similarity(相似度) | 观察 | ✅ 成熟 | `cosine(v1, v2) → float; CKA(H1, H2) → float` |
| Subspace(子空间) | 观察 | ✅ 成熟 | `pca(hidden, n_comp) → components, variance` |
| Attribution(归因) | 分析 | ⚠️ 部分可用 | `attribution(model, input, output) → contributions` |
| Ablation(消融) | 因果 | ⚠️ 需修复 | `ablate_head(layer, head) → modified_output` |
| Patching(替换) | 因果 | ✅ 成熟 | `patch_hidden(layer, tensor) → modified_output` |
| Path Tracing(路径) | 因果 | ❌ 未实现 | `trace_path(source, target) → causal_path` |

#### 评估指标（4维 → 每个实验必须输出）

| 指标 | 含义 | 测量方法 | 最低标准 |
|------|------|---------|---------|
| Separability | 可分性 | probe accuracy, AUC | >0.8 (vs随机基线) |
| Causality | 因果性 | switch_rate, prob_lift | switch_rate>0.3 |
| Stability | 稳定性 | 跨模板/跨样本一致性 | cos>0.5 |
| Generalization | 泛化性 | 跨模型一致性 | 3模型一致 |

#### ModelInterface — 模型访问

```python
class ModelInterface:
    forward(text)           # 前向传播
    get_hidden(layer)       # 获取某层残差流
    patch_hidden(layer, tensor)  # 替换某层残差流
    ablate_head(layer, head)     # 消融某注意力头
    get_attention(layer)    # 获取注意力模式
    get_head_output(layer, head) # 获取单个注意力头输出（修复后）
```

#### DataEngine — 数据生成

```python
class DataEngine:
    generate(config)   # 按配置生成实验数据
    mutate(sample)     # 变异样本（如换词/换模板）
    label(sample)      # 标注样本（类别/属性/逻辑）
```

**数据配置规范**：
```json
{
    "concept_pool": ["apple", "dog", "car", ...],   // 概念池（≥60个）
```


#### Operations — 操作原语


## 六、动态轨迹系统

### 6.1 状态演化方程

```
h_{l+1} = h_l + Attn(h_l) + FFN(h_l)
```

等价于（从INV-6已确认）：

```
gap(l+1) = gap(l) + delta(l)    # 加法传播定律
```

### 6.2 三种时间维度

| 时间维度 | 含义 | 对应物理量 | 当前工具 |
|---------|------|-----------|---------|
| 层时间 | Layer 0 → Layer N 的逐层传播 | 残差流 h_l | ✅ get_hidden(l) |
| Token时间 | 序列生成过程中的token位置 | 位置编码 + 因果注意力 | ✅ forward(text) |
| Attention时间 | 信息从源头token流向目标token的路径 | 注意力权重矩阵 | ⚠️ 需逐头追踪 |

### 6.3 轨迹记录协议

```python
# 记录一个token在所有层的轨迹
trajectory = []
for layer in range(n_layers):
    h = model.get_hidden(layer, token_pos)
    trajectory.append({
        "layer": layer,
        "vector": h,                    # d_model维向量
        "norm": float(h.norm()),
        "cos_with_inject": float(cos(h, h_inject)),
    })
```

**轨迹数据结构规范**：
```json
{
    "token": "apple",
    "template": "X is a Y",
    "trajectory": [
        {"layer": 0, "norm": 12.5, "cos_with_inject": 0.95},
        {"layer": 18, "norm": 156.3, "cos_with_inject": 0.12},
        {"layer": 35, "norm": 412.7, "cos_with_inject": 0.01}
    ]
}
```

## 十二、当前最大瓶颈与第一性原理分析

### 12.1 最大瓶颈：7%→60%的鸿沟

当前填充率只有7%，距离临界点60%有8倍差距。核心原因：

1. **因果工具不足**：注意力头消融方法有bug，路径追踪未实现
2. **维度灾难**：d_model=2560-4096，但样本量只有24-192
3. **多维度未解耦**：template/category/word方差非正交，无法精确分解
4. **缺乏统一语言**：不同Phase用不同概念集，无法交叉验证

### 12.2 第一性原理：语言是什么？

从逆向工程的角度，语言的本质是：

```
语言 = 语义空间 + 子空间结构 + 动态轨迹 + 计算路径
```

- **语义空间**：概念在高维空间中的编码（KN-1a，已部分确认）
- **子空间结构**：知识/逻辑/语法在不同正交子空间中编码（INV-13，已确认）
- **动态轨迹**：概念在层间的传播路径（INV-6加法传播定律，已确认）
- **计算路径**：从语义到输出的因果链（深层锁定机制，未破解）

### 12.3 突破瓶颈的关键洞察

**核心洞察**：PCA主成分由category驱动（INV-25），但概念替换干预在浅层最有效（INV-27）。这意味着：

1. **浅层**：概念信息以"可替换"的方式编码 → 残差流包含"概念变量"
2. **深层**：概念信息被"锁定"为不可替换 → 存在某种"纠错机制"
3. **中间层**：template热点是"语法加工厂" → 句式处理专门层

**如果破解深层锁定机制，就能理解语言生成的核心控制逻辑——这可能是破解数学原理的关键突破口。**
