## 1. ICSPB (分层路径丛理论)
**Inventory-Conditioned Stratified Path-Bundle Theory**
- **原理**：将认知表征视为**纤维丛（Fiber Bundle）**结构。底层流形（Base Manifold）代表基本的感知状态，而纤维（Fiber）则代表在该状态下依库存（Inventory）而定的高阶语义可能。
- **核心**：强调 路径依赖（Path-dependency），即同一状态在不同演化路径下的语义截面是不同的。它解决了符号在连接主义背景下的动态定位问题。

## 2. UCESD (统一因果涌现与结构动力学)
**Unified Causal Emergence and Structural Dynamics**
- **原理**：探讨从微观神经元动力学到宏观因果实体的**粗粒化（Coarse-graining）**过程。
- **核心**：定义了因果效力（Causal Efficacy）的数学度量，证明了宏观因果在特定拓扑约束下可以具备比微观层级更强的预测能力和干预力。它是解决脑侧因果闭环的理论动力源。

## 3. CPT (因果路径理论)
**Causal Path Theory**
- **原理**：专注于在稠密的神经连接中识别并提取**唯一的、非冗余的因果流图**。
- **核心**：通过因果干预判据，修剪掉所有不具备唯一语义贡献的幻觉路径，确保系统的每一个输出都能回溯到确定的长链 Trace。

## 4. GUIT (几何通用信息论)
**Geometric Universal Information Theory**
- **原理**：利用**现代微分几何（Fisher Information Metric）**重新诠释经典信息论。
- **核心**：将表征压缩视为流形上的曲率收缩（Curvature Contraction）。它规定了智能系统的熵减过程必须符合流形拓扑的平滑约束，从而为 AGI 提供了一套基于几何的信息处理极限（Boundaries）。

## 5. UGMT (统一几何流形拓扑)
**Unified Geometric Manifold Topology**
- **原理**：这是整个体系的**总线与归宿**。
- **核心**：它通过拓扑流形的形式，将 ICSPB 的结构、UCESD 的动力学、CPT 的路径、GUIT 的信息测度统一为一套单一的、自洽的几何语言。在这个层面上，智能不再是算法，而是高维流形本身的演化性质。

family patch
原理：同一概念族会共享一个稳定底座，比如“苹果/香蕉/梨”先共享水果族的公共底板，再在其上分化。它解释的是“为什么概念不是孤立 token，而是族群对象”。
动画：显示成一个大而稳定的半透明主团块，周围是同族小节点围绕它缓慢呼吸。不同族群用不同主色，族内共享底座始终不消失。
DNN 分析结果：适合承接 static / subspace_geometry / category compare，展示同类概念神经元是否聚成稳定 patch。

concept section / concept offset
原理：具体概念不是整个 patch 本体，而是 patch 上的一个局部截面；语义微调表现为截面内偏移，不是全空间重建。
动画：在大团块里切出一片会发光的局部薄层，切换“苹果 -> 青苹果 -> 烂苹果”时，这片截面平滑滑移，而不是整团乱跳。
DNN 分析结果：适合 subspace_geometry / feature_decomposition / counterfactual，看最小语义改动是否表现成局部偏移。

attribute fiber
原理：颜色、大小、甜度、材质这类属性更像沿对象截面伸出的纤维方向，可组合、可叠加、可局部拆分。
动画：从概念截面向外长出几束细长发光纤维，不同属性方向不同颜色；调节组合权重时，纤维长度和亮度连续变化。
DNN 分析结果：适合 feature_decomposition / compositionality，展示属性轴是否真能线性或近线性叠加。

relation-context fiber
原理：关系和上下文不是额外标签，而是把对象接入语境网络的纤维。它决定“苹果 在 树上”和“苹果 在 电脑里”为什么会走不同路径。
动画：对象之间出现定向发光桥，桥的弯曲、粗细、流速代表关系强度与语境方向；换上下文时，桥会改道但不会整体坍塌。
DNN 分析结果：适合 cross_layer_transport / dynamic_prediction / counterfactual，看关系信息是否沿稳定路径传播。

admissible update
原理：不是任何更新都合法。好更新应当是局部、受限、不会把旧结构整体冲垮的更新。
动画：当发生学习或干预时，只允许局部节点和局部纤维闪烁重组，主 patch 保持稳定；非法更新则表现为整团抖动、颜色污染、结构塌陷。
DNN 分析结果：适合 robustness / causal_intervention / training-like probes，看哪些变化是“可容许”的。

restricted readout
原理：模型输出不是从全网平均读出，而是从少数受限热点、局部子回路、读出通道产生。
动画：场景里大部分节点保持背景态，只有少数热点节点被逐步聚焦，最后一束细亮光从热点汇到输出端。
DNN 分析结果：适合 causal_intervention / minimal_circuit / dynamic_prediction，看真正决定输出的局部子集是谁。

stage-conditioned transport
原理：同一个输入在不同阶段走不同运输路径。早期像感知对齐，中期像结构整理，后期像读出准备。
动画：把层间主路径分成几段不同颜色的轨道，时间推进时主亮点按阶段切换轨道，不是一路直冲。
DNN 分析结果：适合 dynamic_prediction / cross_layer_transport，看层间热点是否按阶段迁移。

successor-aligned transport
原理：推理和 next-token 不是随机跳，而是朝“后继状态”对齐的运输。它描述的是“为什么后面的 token 是从前面的轨迹里长出来的”。
动画：当前 token 是一个亮核，它沿一条逐渐收束的路径向后推进，路径末端出现后继亮点，旧点衰减，新点接棒。
DNN 分析结果：适合 dynamic_prediction / counterfactual / causal_intervention，看后继输出是否沿稳定对齐轨迹生成。

protocol bridge
原理：内部编码最终必须进入任务、工具、动作、接口层，否则只是内部漂亮结构，没有功能闭环。
动画：内部 patch/section/fiber 最后汇聚到几个规则化端口，端口再连接外部任务板、工具图标或输出槽；桥接成功时端口稳定发亮，失败时闪烁断流。
DNN 分析结果：适合 minimal_circuit / unified decode / system summary，看内部结构如何变成可执行协议。

大规模概念 atlas 建模

family patch 基底识别
对每个 family 做局部 SVD/PCA/chart fit，稳定识别 b_(f)，验证它是不是低秩局部基底，而不是偶然聚类。

concept offset 稀疏展开识别
直接计算 delta_c = z_c - b_(f_c)，再验证
delta_c ~= sum_k a_(c,k) u_(f_c,k) + epsilon_c
看 family 内局部轴 u_(f_c,k) 是否复用、系数 a_(c,k) 是否稳定、残差 epsilon_c 是否持续很小。

不只看静态偏移，还要写出
adaptive_offset_(t+1) = F(adaptive_offset_t, routing_t, novelty_t, stabilization_t, replay_t)
也就是新概念写入、回放固化、稳定保持、遗忘风险如何共同控制 offset。

跨 family scaffold 分离
把 family-local offset 和全局 recurrent scaffold 分开，不然你会把共享全局轴误判成 family 内偏移。

预测闭环验证

新概念会落在哪个 family patch

最小语义改动会造成什么偏移变化
预测不了，就还只是拟合，不算破解。
多模态与多脑区一致性验证
检查视觉、语言、听觉、动作以及不同脑区是否共享同一套 patch + offset 机制，只是参数化不同。

阶段一 survey（普查池）
目的是先拿大样本结构地图，不追求单词极深解释。核心做法是给很多词抽稀疏签名、层分布、家族邻域，先回答“哪些词在模型里看起来像同一家族”。

阶段二 focus（聚焦池）
目的是从大样本里挑真正值得深挖的词族。核心做法是围绕 anchor（锚点词）、challenger（挑战词）、support（支撑词） 去建立家族内部对照，让后面不只是看单点分数，而是看家族内排异和稳定性。

阶段三 causal closure（因果闭合）
目的是验证某些神经元集合是不是“真的在起作用”。核心做法是做 ablation（消融），看去掉共享神经元或个体神经元后，家族边距和类别读出会不会掉。这里第一次从相关性进入因果性。

阶段四 minimal circuit（最小电路）
目的是把“很多相关神经元”压缩成“尽量小但还有效的子集”。核心做法是从阶段三拿到的大集合里做递减搜索，比较子集和随机对照，找最小还能保住结构边距和读出边距的候选子电路。

阶段五 readout-coupled search（读出耦合搜索）
目的是进一步把子电路压到 1-3 个神经元级别，看哪些微团簇既影响内部结构，又影响最终类别读出。这里本质上是在找“最接近精确编码”的微电路见证，但现在也正因为口径更严，所以最容易暴露出类别词污染、公共尾带塌缩、实例词难闭合这些真实问题。




广支撑底座
它指的是：大量神经元都参与，但大多数只是低密度、低优先级参与。它像地基，负责“能表示很多东西”，不是直接负责“最后输出哪个东西”。如果没有它，系统没有足够大的表示空间；但只看它，也解释不了真正的功能分工。

长期分离前沿
它指的是：虽然底座很广，但真正高质量、高决定性的核心不会很快混在一起，而是沿风格、逻辑、句法等方向长期保持分离。这个“前沿”就是最有判别力、最有控制力的那一圈支撑。它的意义是：语言系统不是平均搅成一锅，而是“底座共享，核心分工”。

晚层骨架迁移
它指的是：稳定的概念骨架不是在最前面就完全定死，而是随着层数加深，逐步往更晚层主脊收束。这里的“迁移”很重要，因为它说明骨架不是静态标签，而是一个逐步收口过程。现在的数据支持：有利的逻辑骨架，往往出现在更晚的主层带，而不是停在早层。

中段句法筛选
它指的是：句法的关键作用不是“给内容加料”，而是在高质量前沿的中段形成一片筛选带，把不符合结构约束的候选持续淘掉。所以它更像筛子，不像加法器。我们看到句法正项在密度前沿中段最稳，这就是它叫“中段句法筛选”的原因。

晚窗口闭包
它指的是：真正把原型、实例、关系、约束一起收束成最终输出的关键过程，大多发生在句尾前几个词元（token，词元）窗口，而不是最后一个词本身。最后一个词更多像读出结果，真正的闭包早一点就开始完成了。

把这五个放在一起，当前最简洁的系统图像是：
先有 广支撑底座 提供大空间，
再由 长期分离前沿 保持核心分工，
然后 晚层骨架迁移 立住概念主骨架，
接着 中段句法筛选 持续淘掉坏候选，
最后在 晚窗口闭包 里完成真正收口。








当前项目对语言原理的分析，已经不是把语言当成单独对象看，而是把它看成统一状态系统在输出层和时间轴上的投影。现在最清楚的一条语言主链是 q / b / g，也就是 条件门控场（conditional gate field）、上下文偏置张量（context bias tensor）、门控路由概率（gated routing probability） 这条上下文协变链。Stage72（第七十二阶段） 已经把这条链推进成了可测方程，所以语言现在在项目里不再只是“现象描述”，而是一个可以被方程约束的投影面。

它和大脑编码机制的关系，现在也不是松散类比了。项目当前已经比较明确地认为：这些语言变量并不是纯抽象符号，而是在脑编码层有对应实现。最明显的进展是 Stage77（第七十七阶段） 和 Stage78（第七十八阶段） 已经把路由尺度问题做清楚了：路由主导层不是单神经元独立开关，而是 distributed network（分布式网络） 主导的门控场。所以，大脑编码机制在这里扮演的是“统一变量的物理实现层”，不是语言理论的旁证。

它和智能的数学原理之间的关系，则更进一步。项目现在不是把智能当成高层名词，而是把它看成这些统一变量在任务约束下形成的稳定性、恢复性、迁移性和组合性。Stage79（第七十九阶段） 已经把 forward routing（前向路由） + backward repair（反向修复） 写成计算链，Stage81（第八十一阶段） 把两者放进同一个统一环里，Stage82（第八十二阶段） 又把最难的 novelty_generalization（新颖泛化） 做出了首个可执行修复。也就是说，现在项目已经能把语言投影、大脑编码落地、智能闭合三件事，压到同一个状态系统里来读。
