# AGI GPT5 语言主线左侧控制面板重构方案

## 1. 当前问题

当前左侧控制面板主要由两层组成：

1. `App.jsx`
   - 负责左上主面板容器
   - 负责 `inputPanelTab` 切换

2. `StructureAnalysisPanel.jsx`
   - 负责 `activeTab`
   - 负责算法表单
   - 负责发起分析请求

当前结构更像：

`系统类型 -> 算法分类 -> 参数表单 -> 运行`

这套结构适合“分析工具箱”，不适合 `AGI_GPT5_LANGUAGE.md` 的语言主线，因为语言主线真正的工作对象不是算法，而是：

1. 五层测试体系
2. 研究对象组
3. 任务语义组
4. 结构主线
5. 风险与保真缺口

因此左侧控制面板必须从“工具入口”改成“研究导航入口”。

---

## 2. 重构目标

左侧控制面板改造后，应让用户在 `10` 秒内完成以下判断：

1. 当前在看哪一层研究拼图。
2. 当前在看哪个对象组或任务组。
3. 当前主 3D 空间解释的是哪条结构链。
4. 当前用哪个模型和哪个阶段口径在看。
5. 当前最大的硬伤是什么。

一句话目标：

**让左侧面板从“选工具”升级成“选研究对象、选结构链、选风险焦点”的中枢。**

---

## 3. 新的面板信息架构

建议左侧控制面板改成五段式结构。

### 3.1 第一段：研究层切换

替代当前“按算法组切换”的上半部分。

建议做成一级导航：

1. `静态编码层`
2. `动态路径层`
3. `结果回收层`
4. `传播编码层`
5. `语义角色层`

这 5 个入口直接对应 `AGI_GPT5_LANGUAGE.md` 的五层测试体系。

作用：

- 决定当前主 3D 空间的解释维度
- 决定右侧信息面板显示什么摘要
- 决定中下方可选图层和指标

### 3.2 第二段：研究对象选择器

建议放在研究层下方，作为二级导航。

建议包含 3 组：

1. `对象组`
   - 水果
   - 动物
   - 工具
   - 品牌
   - 文件/代码对象

2. `任务组`
   - 翻译
   - 修改
   - 重构
   - 长链
   - 高竞争

3. `角色组`
   - 对象
   - 属性
   - 位置
   - 操作
   - 约束
   - 结果

作用：

- 决定 3D 场景中的主体对象
- 决定路径桥图显示哪个任务入口
- 决定差分图和角色图的筛选范围

### 3.3 第三段：结构主线开关

这块是左侧面板的核心。

不建议继续用单选 tab，而建议用多选图层开关：

1. `共享基底`
2. `局部差分`
3. `路径放大`
4. `语义角色`
5. `来源保真`

推荐默认值：

- 默认勾选：
  - `共享基底`
  - `局部差分`
  - `路径放大`
- 当研究层切到 `语义角色层` 时，默认打开 `语义角色`
- 当研究层切到 `传播编码层` 或 `结果回收层` 时，默认打开 `来源保真`

作用：

- 决定 3D 场景叠加哪些解释层
- 决定哪些指标卡出现在信息面板中

### 3.4 第四段：模型与阶段口径

建议放在中下区域，作为分析口径控制区。

建议包含：

1. `模型选择`
   - `GPT-2`
   - `Qwen`
   - `DeepSeek`

2. `阶段选择`
   - `Stage257`
   - `Stage258`
   - `Stage260`
   - `Stage262`
   - `Stage265`
   - 以及后续阶段

3. `对照模式`
   - 单模型
   - 跨模型
   - 单阶段
   - 阶段回放

作用：

- 决定指标和图层的数据来源
- 决定是否显示迁移强弱与主核筛选

### 3.5 第五段：固定硬伤板

这一块必须常驻底部，不建议折叠。

建议固定显示：

1. `天然来源保真低`
2. `同类高竞争边界脆弱`
3. `修复强于原生闭合`
4. `品牌义边界弱`
5. `跨模型硬主核仍少`

作用：

- 防止客户端只展示亮点
- 保证当前主矛盾一直在主视觉中

## 4. 建议的状态模型

当前面板的核心状态主要是：

1. `inputPanelTab`
2. `systemType`
3. `structureTab`
4. 各类表单 state

如果按语言主线重构，建议新增一个统一状态对象：

```js
const [languageWorkspace, setLanguageWorkspace] = useState({
  researchLayer: 'static_encoding',
  objectGroup: 'fruit',
  taskGroup: 'translation',
  roleGroup: 'object',
  structureOverlays: ['shared_base', 'local_delta', 'path_amplification'],
  modelKey: 'gpt2',
  stageKey: 'stage260',
  compareMode: 'single_model',
  riskFocus: 'fidelity',
});
```

建议原则：

1. `activeTab` 不再做主导航，只保留为底层渲染映射
2. 新的主导航使用 `languageWorkspace`
3. 旧算法 tab 只作为“高级分析工具”存在

## 5. 组件拆分建议

建议把当前 `StructureAnalysisControls` 拆成语言主线专用的组件树。

### 5.1 顶层容器

新增：

- `LanguageResearchControlPanel.jsx`

职责：

- 承接整个语言主线左侧面板
- 统一管理 `languageWorkspace`
- 组合下方所有子组件

### 5.2 一级导航组件

新增：

- `LanguageResearchLayerTabs.jsx`

职责：

- 渲染五层测试体系入口
- 切换主研究层

### 5.3 对象与任务选择组件

新增：

- `LanguageSubjectSelector.jsx`

职责：

- 渲染对象组、任务组、角色组三联选择器

### 5.4 结构图层开关组件

新增：

- `LanguageOverlaySwitches.jsx`

职责：

- 控制共享基底、局部差分、路径放大、语义角色、来源保真等图层

### 5.5 模型与阶段口径组件

新增：

- `LanguageScopeSelector.jsx`

职责：

- 切换模型、阶段、对照模式

### 5.6 风险与硬伤组件

新增：

- `LanguageRiskBoard.jsx`

职责：

- 固定显示硬伤
- 显示当前风险焦点

### 5.7 高级分析工具折叠区

新增：

- `LanguageAdvancedToolsPanel.jsx`

职责：

- 容纳当前 `StructureAnalysisControls` 里的传统算法工具
- 作为“专家模式”折叠展示

这意味着：

当前的 `circuit / features / causal / manifold / validity` 这些功能不删，但降级为二线入口。

## 6. 文件改造建议

### 6.1 `frontend/src/App.jsx`

建议修改点：

1. 保留左上主 `SimplePanel`
2. 在 `inputPanelTab === 'main'` 时，不再直接渲染 `AppleNeuronControlPanels`
3. 改成渲染新的：
   - `LanguageResearchControlPanel`

建议边界：

- `App.jsx` 只负责：
  - 面板容器
  - 当前系统类型切换
  - 把 `languageWorkspace` 传给主 3D 场景和右侧信息区

### 6.2 `frontend/src/StructureAnalysisPanel.jsx`

建议修改点：

1. 不再承担“语言主线的一级控制面板”
2. 保留为“高级分析工作台”
3. 内部原有 `activeTab` 机制继续保留，但不再是左侧面板的第一层入口

建议新的定位：

`StructureAnalysisPanel.jsx` 从“主控制面板”变成“高级分析抽屉”。

### 6.3 `frontend/src/config/panels.js`

建议新增配置：

1. `LANGUAGE_RESEARCH_LAYERS`
2. `LANGUAGE_OBJECT_GROUPS`
3. `LANGUAGE_TASK_GROUPS`
4. `LANGUAGE_ROLE_GROUPS`
5. `LANGUAGE_STRUCTURE_OVERLAYS`
6. `LANGUAGE_MODELS`
7. `LANGUAGE_STAGE_PRESETS`
8. `LANGUAGE_RISK_ITEMS`

这样未来扩展阶段和对象时，不用继续把静态数据写死在组件里。

### 6.4 `frontend/src/locales.js`

建议新增：

1. 左侧语言主线面板标题
2. 五层测试体系文案
3. 结构主线文案
4. 风险板文案

## 7. 左侧面板的推荐布局

建议从上到下如下：

1. `研究层导航`
2. `对象 / 任务 / 角色选择`
3. `结构图层开关`
4. `模型 / 阶段 / 对照模式`
5. `当前结论摘要卡`
6. `当前硬伤板`
7. `高级分析工具（折叠）`

其中：

- `当前结论摘要卡` 建议根据当前研究层动态变化
- `高级分析工具` 默认收起

---

## 8. 3D 主空间的联动规则

左侧面板改了之后，3D 主空间要跟着变。

建议联动规则如下：

### 8.1 研究层决定基础场景

1. `静态编码层`
   - 默认显示对象群聚、共享底盘、差分热点

2. `动态路径层`
   - 默认显示任务入口、路径分裂、方向词桥

3. `结果回收层`
   - 默认显示结果绑定与修复闭合差异

4. `传播编码层`
   - 默认显示来源链与断裂点

5. `语义角色层`
   - 默认显示对象、属性、位置、操作、约束、结果的角色图

### 8.2 结构图层决定叠加效果

例如：

1. 勾选 `共享基底`
   - 3D 节点显示家族共享体积

2. 勾选 `局部差分`
   - 高亮差分热点与薄差分位置

3. 勾选 `路径放大`
   - 显示前段小扰动到后段大偏转路径

4. 勾选 `来源保真`
   - 显示来源保真衰减或断裂点

---

## 9. 推荐的迁移顺序

### 第一步：先改配置层

在 `panels.js` 里加入语言主线配置常量。

### 第二步：新增语言主线控制容器

新建：

- `LanguageResearchControlPanel.jsx`

先把左侧面板结构搭起来，不急着全接后端。

### 第三步：把旧算法工具收纳进折叠区

保留原有 `StructureAnalysisControls`，但从主入口降为高级工具。

### 第四步：建立 3D 场景联动

让 `languageWorkspace` 驱动主 3D 空间。

### 第五步：补信息面板与风险板联动

让右侧信息面板同步显示当前研究层、阶段、风险摘要。

---

## 10. 最小可实施版本

如果先只做一轮最有价值的改造，建议优先完成这 4 步：

1. 增加五层测试体系导航
2. 增加对象/任务/角色选择器
3. 增加结构图层开关
4. 增加固定硬伤板

这四步完成后，即使还没把所有数据都接通，左侧控制面板的认知结构也已经从“工具菜单”升级成“研究导航面板”。

---

## 11. 最终一句话方案

**如果主 3D 空间要真正按 `AGI_GPT5_LANGUAGE` 的主线重构，那么左侧控制面板就不该再以算法 tab 为中心，而应改造成以“五层测试体系、研究对象、结构主线、模型阶段口径、当前硬伤”五段组成的研究导航中枢。**
