# AGI_GPT5 界面模块重组草稿 V1

## 1. 目标

这份草稿不是最终界面，而是后续重构主界面的统一参照图。

核心目标只有三条：

1. 控制面板只保留操作，不再承担解释、总结、判断、路线规划。
2. 数据查看形成固定层级，不再把不同粒度的信息堆在一个侧栏里。
3. 战略、理论、长期路线统一放到战略层级路线图，不再回流主工作台。

---

## 2. 模块分工原则

### 2.1 左侧只放操作

左侧面板只允许放：

- 研究层切换
- 场景模式切换
- 筛选条件
- 样本回放控制
- 动画控制

左侧面板不再放：

- 长段说明
- 总览总结
- 性质判断
- 拼图对比
- 理论解释
- 阶段路线

### 2.2 中央只放主场景

中央区域的职责固定为：

- 3D 主视图
- 节点与链路高亮
- 局部回放
- 概念/拼图/样本的空间映射

中央区域不负责文字堆叠和长列表。

### 2.3 右侧只放当前数据

右侧面板只负责“当前正在看的对象”的数据层级查看。

建议固定为四个标签：

1. 当前焦点
2. 层数据
3. 样本回放
4. 资产与证据

### 2.4 底部只放时间轴和状态

底部区域只放：

- `before / bridge / after`（修复前 / 桥链 / 修复后）阶段切换
- 当前风险
- 当前缺口资产
- 当前验证状态

### 2.5 路线图只放战略与理论

战略层级路线图统一承载：

- 语言主线控制台
- 研究总览
- 关键性质
- 验证入口
- 概念关联
- 拼图对比台
- 理论主线
- 阶段任务

## 4. 推荐的界面模块

### 4.1 顶栏

建议固定为全局导航，不堆研究细节。

### 4.2 左侧操作栏

建议压缩为五段以内：

1. 研究层
2. 场景模式
3. 筛选器
4. 样本投射
5. 动画控制

### 4.3 中央 3D 主场景

建议成为唯一视觉中心，不再被小窗口和解释块打断。

### 4.4 右侧数据面板

这里才是多层级数据查看主入口，不再让左侧承担数据详情。

### 4.5 底部时间轴

后续真实轨迹接入后，底部时间轴应成为样本级验证的统一入口。

---

## 5. 当前界面的主要问题

最严格地看，当前界面还存在四个硬伤：

1. 左侧面板仍然承担了过多“信息展示”职责。
2. 数据查看层级还没有完全固定，容易在一个面板里混入不同粒度信息。
3. 中央场景虽然已经变强，但还没有成为绝对主中心。
4. 样本级时间轴和右侧数据面板还没有彻底成型。

---

## 6. 下一阶段的大任务

### 第一阶段：模块重构

- 左侧面板压缩成纯操作栏
- 右侧建立四标签数据面板
- 底部建立统一状态与时间轴

### 第二阶段：数据查看流程打通

- 当前焦点 -> 层数据 -> 样本回放 -> 资产证据

### 第三阶段：真实验证接入

- 把真实 `before_trace`（修复前轨迹）、`bridge_trace`（桥链轨迹）、`after_trace`（修复后轨迹）挂到时间轴和右侧数据面板

---

## 7. 结论

这份草稿图的核心不是“页面更好看”，而是把界面真正变成研究机器：

- 左侧负责操作
- 中央负责观察
- 右侧负责数据
- 底部负责验证节奏
- 路线图负责战略与理论

只有先把这个分工固定下来，后面继续加数据、加验证、加拼图，界面才不会再次退化成信息堆积。
