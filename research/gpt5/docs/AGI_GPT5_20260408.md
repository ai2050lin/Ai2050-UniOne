## 1. “旋转”到底是怎么计算的

### 1.1 基本公式

当前项目里最常见的“旋转角”不是通过恢复完整高维旋转矩阵得到的，而是通过两个向量的夹角得到：

```text
theta = arccos( <u, v> / (||u|| ||v||) )
```

计算过程分四步：

1. 点积：`<u, v> = sum_i u_i v_i`
2. 各自长度：`||u|| = sqrt(sum_i u_i^2)`，`||v|| = sqrt(sum_i v_i^2)`
3. 余弦值：`cos(theta) = <u, v> / (||u|| ||v||)`
4. 角度：`theta = arccos(cos(theta))`

### 1.2 当前项目里的三种主要“旋转”口径

#### A. 局部逐层旋转

比较相邻两层的表示：

```text
theta_l = arccos( <h_l, h_{l+1}> / (||h_l|| ||h_{l+1}||) )
```

它回答的问题是：

- 第 `l` 层到第 `l+1` 层，这一步改了多少方向

它不能直接回答：

- 从最初输入到最终层总共转了多少

#### B. 相对初始层的累计旋转

比较当前层和初始层：

```text
Theta_l = arccos( <h_0, h_l> / (||h_0|| ||h_l||) )
```

它回答的问题是：

- 从 `L0`（初始层）到第 `l` 层，总共偏了多少

它不能直接回答：

- 中间是平滑逐层转过去，还是第一步就几乎转完

#### C. 差异方向旋转

先定义两个上下文的差异方向：

```text
d_l = h_l(A) - h_l(B)
```

再比较相邻层差异方向：

```text
phi_l = arccos( <d_{l-1}, d_l> / (||d_{l-1}|| ||d_l||) )
```

它回答的问题是：

- 某类信息方向是否在层间换道
- 消歧信息是稳定传播，还是被重新编码

它不能直接代表：

- 整个 `hidden state`（隐藏态）都按同样方式旋转

### 1.3 为什么文档里会同时出现两种看似矛盾的话

下面两句话其实并不矛盾：

1. “每层都在旋转”
2. “`>92%`（百分之九十二以上）旋转发生在 `L0→L1`”

原因是它们通常不是一个口径：

- “每层都在旋转”一般指**相邻层夹角非零**
- “`>92%` 旋转发生在 `L0→L1`”一般指**相对 `L0` 的累计总角度**

更准确地说：

- 第一层完成了主方向的大重排
- 后面每层仍有小幅修正，所以局部逐层旋转仍不为零

## 2. `hidden state` 在 layer 中到底对应哪一部分

### 2.1 不是单独对应注意力，也不是单独对应线性变换

当前项目里通过：

```python
model(..., output_hidden_states=True)
```

得到的 `hidden_states` 默认表示的是：

- `embedding`（词嵌入）输出
- 每个 `transformer block`（变换器层块）处理完成后的整层输出
- 最后常常还包括末层归一化后的最终表示

因此，`hidden state` 默认**不是单独等于**：

- `attention`（注意力）子层输出
- `MLP`（多层感知机）子层输出
- 某个线性矩阵参数

### 2.2 更准确的理解

它表示的是：

**一个 token（词元）在该层完整处理结束后的当前表示状态**

这个状态已经混合了：

- 注意力
- `MLP`
- 残差连接
- 层归一化

共同作用的结果。

### 2.3 这对“旋转分析”意味着什么

当当前项目比较：

```text
hidden_states[l] 和 hidden_states[l+1]
```

比较的是：

- 整层表示状态的变化

不是：

- 注意力参数在转
- `MLP` 权重矩阵在转

如果要区分“到底是注意力导致方向变化，还是 `MLP` 导致方向变化”，就必须额外做：

- hook（钩子）消融
- 子模块零化
- patching（激活替换）

这也反过来说明：`hidden_states` 本身默认是**合成后的整层结果**。

## 3. 当前研究进展是否符合纤维丛理论

## 3.1 严格结论

**部分符合纤维丛理论的直觉，但还不构成严格的纤维丛理论。**

### 3.2 当前研究里与纤维丛直觉相似的部分

#### A. 有“基底空间候选”

文本、语义、上下文、任务条件，已经隐约像某种底空间。

#### B. 有“纤维候选”

在固定语义附近，风格、语法、读出细节这些附加自由度，很像附着在同一语义点上的局部自由度。

#### C. 有“联络候选”

层间传播、残差更新、`RMSNorm`（均方根层归一化）校正，看起来像沿着某种规则做平行运输。

#### D. 有“曲率证据”

步角大、路径弯曲、切空间维度变化，已经明显超出普通欧几里得直线传播。

### 3.3 为什么还不能说已经建立了纤维丛理论

还缺少纤维丛最关键的四个严格部件：

1. **底空间没有被严格定义**
2. **纤维没有被严格定义**
3. **局部平凡化没有建立**
4. **过渡函数和结构群没有建立**

所以当前更准确的定位是：

**“流形 + 联络雏形 + 曲率证据”**

而不是：

**“已经证明语言编码就是纤维丛结构”**

### 3.4 若要推进成纤维丛理论，下一步最关键的任务

1. 找到底空间变量：什么才是跨文本稳定的语义主坐标
2. 找到纤维变量：固定语义下风格/语法/任务变化是否形成局部等价类
3. 做局部平凡化验证：局部能否拆成“主变量 × 附加自由度”
4. 做路径依赖 / `holonomy`（闭路回转）实验：不同路径到同一点是否留下不同纤维残差

## 4. “测试体系”里已经做了哪些分析方法

根据 [AGI_GLM5_LANGUAGE.md](/d:/develop/TransformerLens-main/research/glm5/docs/AGI_GLM5_LANGUAGE.md) 的“测试体系”和后续脚本索引，当前方法论已经形成六大类。

### 4.1 几何测绘类

- `Embedding SVD`（词嵌入奇异值分解）
- `PCA`（主成分分析）
- 有效秩
- 参与比
- 旋转角
- 旋转平面
- 曲率
- 路径比
- 切空间捕获率
- 层级结构（Early/Mid/Late）

这类方法擅长回答：

- 表示空间是不是低维主导
- 残差流是不是弯曲路径
- 哪些层发生维度膨胀/收缩

### 4.2 读出与方程类

- hidden→logit 分解
- margin 方程
- delta→logit 链
- 读出子空间分解
- `unembed`（反嵌入）结构分析

这类方法擅长回答：

- 模型最终怎么把内部状态映射成词概率
- 末层读出与中间层传播之间的关系

### 4.3 因果干预类

- activation patching
- 层消融
- 方向移除
- 维度移除
- token leave-one-out
- 跨层传播消融

这类方法擅长回答：

- 改哪里会不会真的打到输出
- 哪些层能修复前面的损伤
- 哪些信息一旦移除就不可恢复

### 4.4 机制拆分类

- `attention vs MLP` 分离
- `RMSNorm` 几何
- 门控幅度、方向、子空间的基元消融
- 旋转机制分解

这类方法擅长回答：

- 现象到底由哪个模块主导
- 是注意力负责，还是 `MLP` 负责，还是归一化负责

### 4.5 操控研究类

- centroid 差方向注入
- Fisher / `LDA` 方向
- `PC` 主成分操控
- 梯度方向操控
- logit 空间直接操控
- prefix 操控
- `PGD` 对抗优化
- `Hessian` 局部曲率分析

这类方法最大的价值不只是“能不能操控成功”，而是：

- 证明哪些操控方法无效
- 暴露表示空间的刚性和修复能力

### 4.6 跨模型比较类

- 四模型基底对齐
- 瓶颈比较
- 信息域比较
- 多模态例外解释
- 训练动力学比较

这类方法擅长回答：

- 哪些规律是语言本身的结构
- 哪些规律只是某个模型或训练方式的副产物

## 5. 这些方法已经破解了什么

### 5.1 已经比较有把握破解的部分

当前这些方法已经足够建立“语言表示是几何动力系统”的大框架，主要收获包括：

1. 语言表示不是词典式存储
2. 残差流像路径，不像静态表
3. `RMSNorm` 不是纯缩放，更像投影/校正算子
4. 中间层传播空间与最终读出空间不是同一个几何对象
5. 后续层会修复很多局部扰动
6. hidden state 加法操控困难，根源是结构刚性，不只是方法差

### 5.2 更准确地说，已经破解的是“宏观层”

已经较稳地拿下的是：

- 宏观几何
- 层间动力学
- 读出结构
- 修复机制
- 操控瓶颈

这说明研究已经从“看几个神经元亮不亮”推进到了：

**状态空间几何 + 动力学 + 因果干预**

## 6. 为什么现在还不能说“已经破解语言背后的数学结构”

还差四个根本缺口：

### 6.1 概念级编码还没定位

目前知道“有几何结构”，但还不知道：

- “苹果”在空间里怎么表示
- “红色”“甜味”是独立方向，还是概念向量的组成部分
- 抽象层级（苹果→水果→食物→物体）有没有几何对应

### 6.2 逻辑级计算过程还没还原

当前主要测的是：

- 几何形状
- 扰动响应
- 读出关系

但还没真正拆开：

- 条件推理
- 量词推理
- 组合属性
- 深度逻辑链条

### 6.3 预测能力仍然偏弱

现有很多结果属于：

- 描述很强
- 解释很强
- 但外推预测还不够稳

也就是：

**描述方程已经很丰富，因果生成方程还不够强。**

### 6.4 神经元级机制还没打开

目前大多数方法分析的是：

- 整体 `hidden state`
- 子空间
- 几何统计量

还没有真正深入到：

- 选择性神经元群
- 最小因果回路
- 概念在神经元级别的可解释构造

## 7. 还能尝试哪些方法

如果目标真的是“破解语言背后的数学结构”，我认为最值得补的新增方法有八类。

### 7.1 概念-属性因果定位

- 为单个概念及其属性构造最小对照集
- 用 patching、维度归因、方向归因定位它们
- 判断属性是独立方向，还是概念内部组合成分

### 7.2 逻辑过程追踪

- 设计严格控制的条件推理、量词推理、指代解析、组合属性任务
- 逐层定位“推理在哪一层发生”
- 找推理路径而不是只看最终结果

### 7.3 局部坐标图方法

- 对某个语义邻域做局部线性化
- 估计局部主曲率
- 拟合局部 `chart`（坐标图）
- 验证是否存在稳定主坐标与附加自由度

### 7.4 路径依赖 / `holonomy` 实验

- 让模型沿不同语义路径到达同一结论
- 检验表示是否留下不同纤维残差
- 这是验证纤维化结构非常关键的一步

### 7.5 稀疏字典 / 稀疏自编码器

- 把整体 `hidden state` 分解成可解释原子
- 看概念是单轴、稀疏组合，还是高度分布式叠加

### 7.6 动态系统识别

- 估计局部 `Jacobian`（雅可比矩阵）
- 做 `Koopman`（库普曼）近似
- 做状态转移谱分析
- 判断层传播能否被低维控制系统逼近

### 7.7 信息分解方法

- 做 `PID`（部分信息分解）
- 做条件互信息
- 分离哪些量贡献语义、哪些量贡献风格、哪些量贡献语法

### 7.8 跨模型语义同构检验

- 不只比 `PCA` 对齐
- 而是比局部邻域关系
- 比因果方向
- 比概念间可运输性是否同构

## 8. 下一阶段最优先的大任务顺序

如果目标是“破解语言背后的数学结构”，最值得按这个顺序推进：

1. **概念-属性-抽象层级图谱**
2. **逻辑推理逐层追踪**
3. **局部坐标图 + 路径依赖实验**
4. **跨模型同构检验**

这个顺序的意义是：

- 先回答“语言里到底有什么对象”
- 再回答“这些对象怎么被计算”
- 再回答“这些计算是不是流形/纤维化结构”
- 最后回答“这是不是语言本身的结构，而不是模型偶然学出来的”

## 9. 最严格的总判断

当前测试体系已经明显超过了普通可视化或简单探针水平，已经进入：

**几何测绘 + 因果打击 + 动力学测量 + 操控判伪**

这个层级。

但如果目标是“破解语言背后的数学结构”，现在最稳的说法仍然应该是：

1. 已经比较清楚地摸到了语言表示的宏观几何骨架
2. 已经知道模型内部存在路径、投影、修复、刚性这些结构性规律
3. 但还没有把概念、属性、逻辑、语法这些微观语义对象真正拆成可识别的数学坐标系

因此，当前阶段最准确的定位不是：

**“已经破解语言的数学结构”**

而是：

**“已经建立了破解语言数学结构所需的宏观动力学框架，接下来必须转向微观编码图谱与逻辑计算机制。”**
