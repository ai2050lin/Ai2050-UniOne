# AGI_GPT5_LANGUAGE

## 0. 当前研究原则

当前语言研究主线正式采用以下原则，这一部分放在文档最前面，作为后续所有测试与分析的总前提：

1. 不预设现有数学体系足以描述该结构  
   当前更合理的判断是：大语言模型（Large Language Model，大语言模型）内部的语言结构，很可能超过现有成熟数学工具的直接表达能力。

2. 不再用公式给结构画图例  
   后续研究不再把“写出一个看起来整齐的公式”当成优先目标。任何公式、分数压缩、线性组合，都只允许作为阶段性记录工具，不允许被当成最终理论本体。

3. 只通过测试去寻找更多拼图  
   研究重点是不断通过真实测试找到更多稳定拼图，包括共享核、差分边界、纤维复用、路径选路、结果回收、来源痕迹、多实体竞争、跨模型共同核。

4. 先补齐拼图板，再反向逼出新理论  
   只有当拼图块足够完整、连接关系足够稳定时，新的数学结构才有可能真正浮现出来。现在的目标不是“提前写理论”，而是“先把结构找全”。

5. 以跨模型稳定性作为高优先级约束  
   `GPT-2（模型）`、`Qwen（通义千问）`、`DeepSeek（深度求索）` 中共同稳定出现的拼图块，优先级高于单模型里看起来很漂亮但不稳定的局部现象。

6. 以缺口定位替代过早闭环  
   当前每一轮研究的重点，不是宣布闭式完成，而是明确：
   - 哪些拼图块已经稳定
   - 哪些拼图块只是过渡态
   - 哪些地方仍然是硬缺口

一句话概括当前原则：

**先完成拼图，再破解结构；先积累事实，再倒逼新数学。**


## 1. 测试方案

这份文档的目标，不是简单记录几个语言实验，而是建立一套可以持续挖掘 `GPT-2（生成式预训练变换器二代）`、`Qwen3（通义千问三代）`、`DeepSeek（深度求索）` 三个模型语言原理的系统方案。

当前最核心的研究问题是：

**模型里的语言能力，究竟是哪些更基础的动力学变量共同作用的结果。**

为了逼近这个问题，当前测试方案不是“只看一个分数”，而是把语言拆成五层连续链条：

1. 词表静态层  
   看词嵌入、词类、尺度带、语义家族，回答“模型把词先编码成了什么样的静态结构”。

2. 句中定锚层  
   把词放入句子里，观察目标词在早层是否会出现稳定定锚，回答“模型在真正处理句子时，先抓住了什么”。

3. 动态选路层  
   重点观察副词、动词位、结果位，回答“上下文是否真的改变了内部选路，而不只是改变最后输出分布”。

4. 语篇传播层  
   重点看重提、回指、省略、跨句续接，回答“语言能力是否能跨句保持、回返与补救”。

5. 变量压缩层  
   把前面观测到的现象压成候选变量：
   - `a`：局部激活密度 / 早层定锚强度
   - `r`：回返一致性 / 回指与省略恢复
   - `f`：续接流 / 后层聚合与补救
   - `g`：门控路由 / 动词位与结果位选路
   - `q`：条件门控场
   - `b`：上下文偏置

### 1.1 测试为什么这样设计

如果只做“词表相似度”，看不到动态语言机制。

如果只做“句子输出结果”，又很难知道机制发生在哪一层。

如果只看一个模型，很容易把模型局部现象误当成普遍规律。

所以当前方案坚持三个原则：

1. 先静态，后动态  
   先知道词如何被编码，再看词进入句子后如何被激活、选路和传播。

2. 先局部，后语篇  
   先看单句中的早层定锚与中层选路，再看跨句回返和补救。

3. 先单模型，后跨模型  
   先在 `GPT-2` 找出候选机制，再去 `Qwen3` 和 `DeepSeek` 检查哪些能迁移。

### 1.2 当前测试数据是如何构造的

当前数据不是随便写几条句子，而是按三种层次构造：

#### 1.2.1 词级数据

用于词嵌入全词扫描，主要回答：

- 哪些词更像名词、动词、形容词、副词、功能词
- 哪些词更偏 `micro（微观）`、`meso（中观）`、`macro（宏观）`
- 哪些词更像桥词

这一步主要产出：

- 词类标签
- 尺度带标签
- 语义家族标签
- 有效编码分数

#### 1.2.2 句级数据

用于句中神经元探针与选路探针，主要回答：

- 名词进入句子后是否会在早层定锚
- 副词是否会改变动词位选路
- 动词位变化是否还能传到句尾结果位

这一步强调最小对照：

- `primary（主样本）`
- `substitute（替换样本）`
- `weaken（弱化样本）`
- `break（破坏样本）`

这样做的原因很直接：

- 如果主样本有效，但替换样本无效，说明模型在读特定机制
- 如果弱化后还能保留，说明机制不是偶然触发
- 如果破坏后断裂，说明这条机制链是真的

#### 1.2.3 语篇级数据

用于复杂语篇传播测试，主要回答：

- 名词第一次出现后，后面重提时是否能被重新拉起
- 代词和省略是否会在后层被补救
- 跨句主题是否能被持续追踪

这里当前采用五类语篇家族：

1. `discourse_remention（语篇重提）`
2. `causal_remention（因果重提）`
3. `nested_memory（嵌套记忆）`
4. `contrastive_remention（对比重提）`
5. `cross_sentence_bridge（跨句桥接）`

### 1.3 当前测试流程

当前完整测试流程可以压成八步：

1. 全词扫描  
   先把词表里的词做静态结构恢复。

2. 词类投影  
   看名词、动词、形容词、副词、功能词分别落在哪个结构带。

3. 副词桥测试  
   找到最有可能通向动态选路的入口。

4. 句中定锚测试  
   检查名词在句中是否形成早层定锚。

5. 动态选路测试  
   检查副词是否改变动词位选路，选路发生在哪一层。

6. 语篇传播测试  
   检查重提、回指、省略、跨句传播是否成立。

7. 变量拟合与反演  
   把这些现象压成 `a / r / f / g / q / b` 候选变量。

8. 跨模型验证  
   在 `Qwen3` 和 `DeepSeek` 上用同口径数据重测，看哪些现象仍成立。

### 1.4 当前推理链条

当前整套语言研究用的推理链条，不是“结果高了所以理论对”，而是：

1. 如果词类在词表层就能分开，说明模型静态编码里已有结构分化。
2. 如果名词进句后会在早层定锚，说明模型不是只在后层做语义合成。
3. 如果副词能改变动词位选路，说明上下文确实在改内部路由，而不是只改最后词概率。
4. 如果名词能跨语篇重提，说明模型内部有持续续接流。
5. 如果回指、省略只能在后层部分补救，说明 `r（回返一致性）` 还没有早层闭合。
6. 如果三模型都复现某个现象，这个现象更接近跨模型主核。
7. 如果某个现象只在 `GPT-2` 很强，在 `Qwen3` 和 `DeepSeek` 弱，就只能算局部规律，不能直接升格成第一性原理。

### 1.5 当前数据工程规模

到目前为止，语言测试已经不再是零散实验，而是一个有数据工程底座的系统。

当前已经落地的两块底座是：

1. `Stage147（第一百四十七阶段）` 机制家族生成器  
   - `6` 个机制家族
   - `4` 个难度层级
   - `4` 种对照类型
   - `384` 条共享机制样本

2. `Stage148（第一百四十八阶段）` 变量可辨识数据包  
   - `6` 个变量包
   - `704` 条变量可辨识样本

再往上，`Stage149（第一百四十九阶段）` 已经把“下一轮该扩什么”自动调度出来，当前最应该先补的是 `b（上下文偏置）`。

### 2.4 推理链条

`GPT-2` 当前的推理链条可以写成：

`词类静态分化 -> 名词早层定锚 -> 副词改变动词位选路 -> 后层继续聚合与补救 -> 复杂语篇中名词可被重提`

- `f（续接流）` 也有证据，因为复杂语篇重提稳定。
- 但 `r（回返一致性）` 和 `b（上下文偏置）` 仍不够强。

### 2.5 测试总结

如果压成一句话，当前 `GPT-2` 的语言结论是：

**`GPT-2` 已经给出了一条比较清楚的语言动力链：词类静态投影清楚，名词句中早层定锚强，副词会在早层改变动词位选路，复杂语篇中的名词重提稳定，但回指、省略、结果闭环和稳定偏置场仍未真正闭合。**

## 5. 跨模型总结

### 5.1 当前跨模型总体结果

当前三模型联合评估的主结果来自 `Stage150`：


### 5.2 当前跨模型稳定的是什么

当前跨模型真正比较稳的，不是整套理论，而是下面三条：

1. `adverb_route_shift（副词动态选路）`  
   三模型都能看到副词改变动词位选路。

2. `conditional_field_fit（条件场重建）`  
   三模型都能高分拟合条件门控场，只是内部主导权重不完全一样。

3. `discourse_remention（复杂语篇名词重提）`  
   三模型都保留了名词跨语篇被重新拉起的现象。

这三条可以看作当前语言理论最稳的跨模型主核。

### 5.3 当前只算过渡成立的是什么

当前只算过渡成立的有两条：

1. `syntax_anchor_band（句中早层定锚带）`
2. `anaphora_repair（回指与省略补救）`

它们的问题不是完全不存在，而是：

- 在 `GPT-2` 比较强
- 在 `Qwen3` 明显变弱甚至断裂
- 在 `DeepSeek` 只有部分保留

所以这两条链还不能直接当作第一性原理主链。

### 5.4 当前最弱的两条链

当前最弱的是：

1. `noun_verb_joint_chain（名词到动词联合链）`
2. `noun_verb_result_chain（名词到动词到结果链）`

也就是说，现在最缺的不是“模型有没有语言现象”，而是：

**名词结构怎样真正进入动作选路，并继续传到结果位。**

这正是当前语言理论最大的闭环缺口。

### 5.5 当前最重要的推理链条

把三模型放在一起后，当前最保守、但最稳的推理链条可以写成：

`词表静态结构存在 -> 副词能进入动态选路 -> 复杂语篇中名词可被重提 -> 条件场可以被重建 -> 但名词早层定锚、回指补救、动作结果闭环还没有成为稳定跨模型主链`

换句话说：

- 我们已经抓住了跨模型都存在的动态语言现象
- 但还没有抓住跨模型都稳定的同一组原生变量

### 5.6 当前最严格的硬伤

最严格地看，当前还有五个硬伤：

1. `b（上下文偏置）` 仍然是最弱代理量  
   说明稳定上下文场还没有被直接测出来。

2. `a（早层定锚）` 不是稳定跨模型主核  
   它在 `GPT-2` 强，在 `Qwen3` 弱，在 `DeepSeek` 半保留。

3. `r（回返一致性）` 还没形成早层闭环  
   现在更像后层补救，而不是早层闭合。

4. `noun -> verb -> result` 链仍然偏弱  
   说明动作闭环还没形成。

5. 三模型联合式里 `g` 权重掉到 `0`  
   说明单模型里亮眼的选路现象，还没稳成跨模型原生变量。

### 5.7 要继续逼近第一性原理，下一步该做什么

当前最合理的下一段任务，不应该再平均铺开，而应该直接围绕最弱点推进三块：

1. `Stage151`：`b（上下文偏置）` 显式化数据包  
   目标是把当前最弱变量单独测出来。

2. `Stage152`：`a -> g` 桥接块  
   目标是追早层定锚怎样真正进入动态选路。

3. `Stage153`：`noun -> verb -> result` 强化块  
   目标是打通当前最弱闭环。

如果这三块能站住，语言理论才有机会从：

- 强现象解释体系

继续推进到：

- 更接近第一性原理的跨模型语言主核

2. 家族神经元束图 里真正可复用束还太少，说明共享和分离依然纠缠。
3. 后层聚合切片 仍然偏弱，说明“后段怎样整合回去”还没彻底站住。
4. 跨模型稳定块仍偏少，说明现在离“真正底层不变量”还有距离。
