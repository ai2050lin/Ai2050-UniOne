# AGI_GPT5_LANGUAGE

## 1. 研究目标

当前主判断是：

深度神经网络的语言能力，不像“背了一本词典”，更像在参数与残差流里形成了一套可计算、可组合、分层分工的隐藏数学结构。

这套结构同时承载：

- 语义
- 语法
- 指代
- 属性绑定
- 最终读出

真正要破解的，不是单个任务，而是这套内部编码机制本身。

## 2. 理论框架

当前最稳的统一框架是：

`route（路由） -> backbone（骨干） -> switch（切换） -> attribute（属性） -> bridge（桥接） -> readout（读出）`

可先写成统一状态方程：

`h_{t}^{l+1} = h_{t}^{l} + R_l(x_{<=t}) + B_l(lemma_t) + S_l(lemma_t, context_t) + A_l(attr_t, context_t) + G_l(B_l, A_l, route_t) + O_l(h_t^l)`

对应含义：

- `R_l`：早层路由项，负责跨词元关系建立，尤其是代词、功能词、句法线索
- `B_l`：名词骨干项，负责公共名词底座和家族复用
- `S_l`：词义切换项，在共享底座上完成多义切换
- `A_l`：属性修饰项，负责颜色、味道、大小等属性写入
- `G_l`：绑定桥接项，把对象骨干与属性修饰绑定到同一对象
- `O_l`：晚层读出项，把内部状态压成最终可输出答案

模块分工暂时可理解为：

- `attention（注意力）` 更像负责路由、搬运、跨词元配对
- `MLP（多层感知机）` 更像负责局部重写、状态压缩、绑定写入
- `residual stream（残差流）` 更像共享工作记忆

## 3. 核心拼图

### 3.1 代词主线

当前最硬的因果证据仍然来自代词：

- `DeepSeek7B` 已出现稳定早层头组骨架
- 核心骨架是 `H:2:0`、`H:2:3`、`H:3:1`
- 最强解释仍是“第 2 层先路由，第 3 层后整合”

这说明语言编码机制不是“哪层亮了”，而是“回路拓扑是什么”。

### 3.2 多义名词主线

多义名词最稳的结论是：

- 不是每个词义一整套独立编码
- 而是“共享名词底座 + 词义切换轴”

`apple` 的水果义和品牌义，就是共享底座上再沿不同方向切换。

这解释了为什么模型既有表达力，又能保持高性能：

- 大量参数复用
- 切换成本低
- 不需要为每个二义性组合单独存档

### 3.3 属性绑定主线

更大属性图谱已经说明：

- 绑定不是整块重写
- 而是“名词骨干 + 属性修饰 + 桥接项”

跨水果、颜色、味道、大小的属性图谱里，这条规律是稳定的，不只是 `apple` 个案。

### 3.4 桥接项的新进展

这是当前最关键的新拼图。

`stage439` 直接打桥接神经元，没有打出强崩塌，说明桥接项不是简单的少量稀疏神经元。

但 `stage443` 的分家族独立探针又给了新信息：

- `DeepSeek7B` 的 `taste（味道）` 绑定只找到弱单头 `H:25:8`
  - `binding_drop（绑定下降） ≈ 0.0122`
  - `heldout_binding_drop（留出集绑定下降） ≈ 0.0163`
  - 不足以支撑强混合回路
- `DeepSeek7B` 的 `size（大小）` 绑定第一次出现稳定混合子回路
  - 子回路：`H:25:12`、`N:26:17120`、`H:23:11`、`H:23:3`、`H:27:19`
  - `binding_drop ≈ 0.0643`
  - `heldout_binding_drop ≈ 0.0521`
  - 达到当前混合回路支持阈值
- `DeepSeek7B` 的 `color（颜色）` 绑定在当前任务上仍存在明显 `CUDA` 数值不稳定，尚未稳定落地

这说明 `G_l` 不是“统一一种形状”的机制，而更像：

- 某些属性家族可被较小的混合子回路承载
- 某些属性家族仍然更分布式
- 某些家族在当前测量任务和数值路径上仍不稳定

也就是说，桥接项开始露出局部可压缩性，但还没有被整体破解。

## 4. 当前最硬结论

当前最稳的理论结论有六条：

1. 语言编码更像分层状态演化系统，不像静态词典
2. 代词机制已显示出“先路由、后整合”的早层回路骨架
3. 多义名词更像“共享底座 + 切换轴”，不是词义独立存档
4. 属性绑定更像“骨干 + 修饰 + 桥接”，不是整块重写
5. 桥接项不是单纯稀疏神经元簇，而是至少部分依赖混合子回路
6. 桥接项在不同属性家族上的稀疏度不同，`size` 比 `taste` 更容易露出局部回路

## 5. 当前硬伤与瓶颈

### 5.1 `G_l` 仍是最薄弱的一环

虽然 `size` 已出现混合子回路，但离“桥接项总理论”还很远：

- `taste` 仍然很弱
- `color` 仍不稳定
- 还没形成统一、可迁移、跨家族稳定的桥接方程

### 5.2 数值稳定性仍是现实瓶颈

本阶段暴露出两个工程硬伤：

- `CUDA error: unknown error`
- `1455（页面文件太小）`

这说明当前实验不仅有理论瓶颈，还有系统执行瓶颈：

- 长进程重复组合搜索容易触发驱动级或库级不稳定
- 多大模型并行加载会压垮页面文件

所以后续实验必须默认采用：

- 分家族
- 分模型
- 串行
- 中途落盘

### 5.3 理论还不是第一性原理

现在的统一方程能组织拼图，但还不能精确预测所有语言行为。

尤其是：

- `G_l` 的具体形式还不清楚
- 各属性家族是否共享同一类桥接变量还未证明
- `Qwen3` 与 `DeepSeek7B` 的统一隐藏变量映射仍不完整

## 6. 下一阶段大任务

下一阶段不要再只做零散扫描，应该直接推进三条主线。

### 6.1 做桥接项的家族级比较

目标不是再问“有没有桥接”，而是问：

- 哪些属性家族可被小回路压缩
- 哪些属性家族天然更分布式
- 为什么 `size` 比 `taste` 更容易露出混合回路

这会把 `G_l` 从单点现象推进到分类理论。

### 6.2 做更稳的系统级实验框架

后续实验默认需要：

- 单模型串行
- 单家族串行
- 每轮中途落盘
- 禁止大模型并发加载

否则理论还没推进，系统先崩。

### 6.3 把三条线压成统一变量

最终目标不是继续堆文件，而是把这三条线压成共同状态变量：

- 代词路由变量
- 名词底座变量
- 词义切换变量
- 属性方向变量
- 桥接绑定变量

只有这些变量能跨模型、跨任务稳定出现，才算开始接近第一性原理理论。

## 7. 阶段性判断

当前最合理的阶段性判断是：

深度神经网络的语言编码，不是词条记忆，而是一个分层、分工、可组合的状态演化系统。

其中：

- 代词主线揭示了路由拓扑
- 多义名词主线揭示了共享底座与低成本切换
- 属性绑定主线揭示了骨干与修饰的组合结构
- `size` 家族第一次给出桥接混合子回路的硬证据

但真正要跨到第一性原理，还必须把 `G_l` 从“局部可见”推进到“统一可写”。
