## FFN增益调制 × 内容方向分解 — 原理详解

### 一、起点：FFN的数学结构

在Gated FFN（如GLM4、Qwen3、DS7B中使用的SwiGLU结构）中，FFN的计算为：

```
FFN(h) = W_down @ [g ⊙ u]
```

其中：
- `u = W_up @ h̃` — **内容候选**，是输入在 `W_up` 投影下的结果，维度为 `n_inter`（约9728~18944）
- `g = σ(W_gate @ h̃)` — **增益门控**，是输入在 `W_gate` 投影后经sigmoid的结果，值域 `(0,1)^n_inter`
- `⊙` 是逐元素乘法（Hadamard积）
- `W_down` 将中间维度投影回 `d_model`

关键洞察：**g 和 u 是同一输入 h̃ 经过两条不同路径产生的**，g 经过了sigmoid非线性，u 是线性投影。

### 二、核心分解：从"绝对量"到"变化量"

考虑两个不同输入 `h_A` 和 `h_B`（对应不同语义，比如"猫"和"狗"），它们产生：

- `g_A, u_A` → `FFN_A = W_down @ [g_A ⊙ u_A]`
- `g_B, u_B` → `FFN_B = W_down @ [g_B ⊙ u_B]`

输出差异为：

```
Δout = FFN_B - FFN_A = W_down @ [g_B ⊙ u_B - g_A ⊙ u_A]
```

令 `Δg = g_B - g_A`，`Δu = u_B - u_A`，`ḡ = (g_A + g_B)/2`，`ū = (u_A + u_B)/2`，展开得：

```
Δout = W_down @ (Δg ⊙ ū)       ← gate项：增益变化 × 平均内容
     + W_down @ (ḡ ⊙ Δu)       ← up项：平均增益 × 内容变化
     + W_down @ (Δg ⊙ Δu)      ← cross项：增益变化 × 内容变化
```

这是一个**数学恒等式**，不依赖任何假设。

### 三、三个项的含义

| 项 | 表达式 | 物理含义 |
|---|--------|---------|
| **gate项** | `W_down @ (Δg ⊙ ū)` | 增益变化对**已有内容**的调制效果 — "同样的信号，放大/缩小了多少" |
| **up项** | `W_down @ (ḡ ⊙ Δu)` | 内容变化在**固定增益**下的投影 — "增益不变，信号本身变了" |
| **cross项** | `W_down @ (Δg ⊙ Δu)` | 增益和内容**同时变化**的耦合效果 |

实验发现 **cross项≈0**，所以主要信号来自gate项和up项。

### 四、增益调制 vs 内容方向 — 两个空间

这是最核心的区分：

#### 1. 内容方向 Δu — 在表示空间

- `W_down @ Δu ≠ 0`（cos约+0.12~+0.41）
- Δu 是**对 W_down 线性可见的**
- 含义：Δu 改变了"生成什么特征" — **改变方向本身**
- 类比：音频信号本身（唱什么歌）

#### 2. 增益调制 Δg — 在控制空间

- `W_down @ Δg ≈ 0`（cos约0.003）
- Δg 是**对 W_down 线性读取不可见的**
- 含义：Δg 改变了"特征被放大/抑制多少" — **改变幅度，不改变方向**
- 类比：混音台上的增益旋钮（调音量，不换歌）

**为什么 W_down @ Δg ≈ 0？**

这是数学必然，不是"无信息"的证据：
- `W_down` 的有效秩 n90 ≈ 88，远小于 n_inter ≈ 9728~18944
- 所以 `W_down` 的零空间维度 ≈ 9640维！
- Δg 的有效维度 n90 ≈ 34-46，完全可能落在 `W_down` 的零空间附近
- **加法性读取头（W_down）看不到乘法性信号（Δg），这是结构性的**

关键：**Δg 的信息通过 ⊙ 运算被"翻译"到了 W_down 可见的空间**：
```
W_down @ Δg ≈ 0        ← 线性读取不可见
W_down @ (Δg ⊙ ū) ≠ 0  ← 乘积非线性读取可见！
```

### 五、增益 × 内容 = 加权特征

两个信号通过 ⊙（逐元素乘法）组合：

```
Δg ⊙ ū = 增益调制 × 内容方向 = 加权特征 → 概念信号
```

这就像：
- **混音台模型**：每个中间神经元是一条音轨
  - `ū` 决定了每条音轨的原始信号（内容）
  - `Δg` 决定了每条音轨的推子位置（增益）
  - `Δg ⊙ ū` 就是推子调节后的混合输出

### 六、层间演化规律

实验发现增益和内容的相对重要性随深度变化：

| 层位 | gate项贡献 | up项贡献 | 含义 |
|------|-----------|---------|------|
| **浅层** | ~9% | ~88% | 增益调制弱，内容方向主导 — "信号本身决定输出" |
| **深层** | 29-52% | 38-51% | 增益调制增强 — "怎么放大变得更重要" |

Swap干预实验也印证：
- 浅层 swap Δg → 类别方向cos=+0.32（改变增益显著影响输出）
- 深层 swap Δu → 类别方向cos=+0.34（深层内容方向效果更大）

### 七、Δg 和 Δu 的弱耦合

`Δg ⊥ Δu` 的 cos ≈ 0.04，看似正交，但需注意：
- 高维空间（n≈9728）中随机向量期望 |cos| ≈ 1/√n ≈ 0.01
- 0.04 只是随机基线的4倍，**不是强正交**
- 正确说法：**增益和内容无强耦合**，但也不是完全独立

### 八、完整信号流

```
输入 h
  ├── W_gate → σ → g (增益，0~1连续)
  │                ↘
  │                 ⊙ → g⊙u → W_down → 输出到残差流
  │                ↗
  └── W_up → u (内容，实数)
  
语义差异：
  Δg(增益调制) ⊙ ū(平均内容) + ḡ(平均增益) ⊙ Δu(内容方向) + ≈0
  
  ≈ 增益调制×内容 + 增益×内容变化
  ≈ "放大/缩小什么" × "信号本身" + "当前增益" × "信号方向变了"
```

### 九、总结

**FFN增益调制 × 内容方向分解** 的本质是：

> FFN通过两条并行路径处理输入——一条产生**内容方向**（"说什么"），一条产生**增益调制**（"说多响"）。两者通过逐元素乘法组合，生成加权特征。增益信号处于控制空间（对线性读取不可见），但通过乘法运算渗透到表示空间，是FFN语义编码的核心机制。
