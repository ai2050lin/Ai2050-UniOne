## TMN 4.0 AGI 测试评估总结

### 修复的Bug（3个）
1. `GlobalSteadyState.sleep_reset` 中 `self.causal_strength` → `manifold.causal_strength`（属性在错误的对象上）
2. `F.normalize` 在1维张量上默认 `dim=-1` 在某些PyTorch版本中导致 `IndexError`，改用手动归一化
3. `fast_sparse_diffuse` 中 `topk(k)` 当 k > 词汇量时崩溃，添加了 `min(k, vocab_size)` 保护

### 测试结果

| 测试项 | 结果 | 详情 |
|--------|------|------|
| 基础构建 | PASS | 所有模块可正常初始化 |
| 预测准确率 | **17.9%** | 100 epoch后仍远低于随机(3.2%)，说明学习机制根本无效 |
| 文本生成 | 运行但输出无意义 | 生成的字符乱序，无法重现训练文本 |
| 反事实推理 | FAIL | 传入token list而非idx list导致编码出错 |
| 全局稳态 | PASS | 自我嵌入范数稳定在合理范围 |
| 开放式生长 | PASS | 新概念可以动态添加 |
| 泛化能力 | 无法泛化 | 只能做最近邻，无法外推 |
| 多模态融合 | PASS | 机制可运行，但多模态权重对结果影响微弱 |

### 核心硬伤（最严格审视）

1. **赫布更新方向错误**：`delta = 0.01 * (ctx - tgt_emb)` 使嵌入**远离**目标而非趋近，这是学习失败的根本原因
2. **无序列建模**：没有位置编码、注意力机制，`encode` 只做平均池化，完全丢失了token顺序
3. **预测本质是最近邻搜索**：`argmax(similarity)` 只能找最相似的已有嵌入，无法生成新组合
4. **因果推理有名无实**：`counterfactual` 只计算点积差，不是 do-calculus 因果推断
5. **蒸馏栈几乎不学习**：初始化为单位矩阵，更新率0.001极小

### 根本结论

**TMN 4.0 不是 AGI**。它本质上是一个"嵌入 + 最近邻搜索"的简单系统，缺少：
- 组合泛化（compositional generalization）能力
- 逻辑推理能力
- 序列依赖建模
- 元学习（meta-learning）
- 自主目标设定

### 突破瓶颈的阶段性大任务

| 阶段 | 任务 | 目标 |
|------|------|------|
| 阶段1 | 修正赫布更新方向 `delta = 0.01 * (tgt_emb - ctx)` | 学习信号方向正确 |
| 阶段2 | 加入位置编码 + 简单序列模型（如RNN/线性注意力） | 理解token顺序 |
| 阶段3 | 将"因果强度"从 `defaultdict` 升级为可学习的因果图（DAG + 有向注意力） | 真正的因果推理 |
| 阶段4 | 引入反向传播或等价优化，替代纯赫布规则 | 学习非线性映射 |
| 阶段5 | 符号-神经混合架构，实现组合泛化 | 从已知概念组合出新的理解 |
