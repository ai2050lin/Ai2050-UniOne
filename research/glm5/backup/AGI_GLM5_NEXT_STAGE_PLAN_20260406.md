### 一、下一阶段的总体判断

当前研究的核心困境是：**我们有了精确的"描述方程"，但没有"生成方程"和"因果方程"。**

具体来说：
- **描述方程**（我们能精确描述发生了什么）：残差流方程误差=0，抵消率56-78%
- **因果方程**（我们能预测如果改变X会发生什么）：**完全缺失**
- **生成方程**（我们能从输入预测输出）：**失败**

这意味着当前研究仍停留在"测量现象"阶段，还没有进入"理解机制"阶段。语言编码背后的数学结构到底是什么，仍然是黑箱。

**关键突破方向**：从"测量"走向"因果干预+反事实推理"。不再满足于"描述delta_l长什么样"，而是要回答"如果我修改delta_l的某个属性，输出会怎么变"。

### 二、六大任务包（P15-P24）

#### 任务包A（P15-16）：Logit方程的因果修复——从cos=3%到可预测

**问题诊断**：margin = cos(d, u)·∥d∥·∥u∥ 方向正确但幅度低估100倍。原因不是"cos小"这么简单——cos=3%时理论margin应该≈0.03×∥d∥×∥u∥，实际margin远大于此。

**假设**：margin的决定因素不只是delta_L与单个unembed方向的对齐，而是：
1. delta_L与**多个相关token**方向的对齐模式
2. unembed矩阵的**非线性结构**（softmax的放大效应）
3. 残差流中的**非消歧分量**对logit的贡献

**实验设计**（Stage662-663）：

**P15（Stage662）**：Logit精确分解
- 不再用 `score_candidate_avg_logprob` 的黑箱估计，而是直接从最后一层hidden state × unembed矩阵计算每个token的logit
- 分解 margin = logit(correct) - logit(incorrect) 为：
  - 消歧分量贡献 = cos(d_L, u_correct - u_incorrect) × ∥d_L∥ × ∥u_correct - u_incorrect∥
  - 共享基底贡献 = cos(h_shared, u_correct - u_incorrect) × ∥h_shared∥ × ∥u_correct - u_incorrect∥
- 目标：量化"消歧分量"和"共享基底分量"对margin的各自贡献比例

**P16（Stage663）**：softmax非线性放大效应
- 计算 logit = W_u @ h_L 的完整logit分布
- 分析 top-10 logit之间的间距分布
- 测试：消歧方向的对齐度是否在softmax之后被放大？
  - 比较 raw_logit差 vs softmax后的概率比
  - 验证假设：小cos_alignment(3-5%)经过softmax后被非线性放大到可区分的程度

**判伪标准**：
- 如果消歧分量贡献 < 10% → margin主要由共享基底决定，当前"消歧方向"框架需要修正
- 如果softmax放大效应 < 2倍 → 问题不在softmax，需要重新审视logit方程形式

**模型**：Qwen3 → DS7B → GLM4（串行）

#### 任务包B（P17-18）：增量差抵消的因果控制——从"浪费"到"可利用"

**问题诊断**：56-78%的增量差被抵消。但这真的是"浪费"吗？还是抵消本身有功能？

**假设**：
1. **噪声过滤假说**：抵消的分量是"随机噪声"，保留的分量是"信号"
2. **维度分离假说**：抵消使不同能力编码在不同方向上，实现正交分离（INV-124）
3. **精度控制假说**：抵消率决定了编码的"锐度"——抵消越多，编码越精确但信息越少

**实验设计**（Stage664-665）：

**P17（Stage664）**：逐层增量差的信号/噪声分离
- 对每层 increment_diff_k 做 SVD，分解为 top-k（信号）和 rest（噪声）
- 计算"信号分量保留率" vs "噪声分量保留率"
- 预测：信号分量被保留（低抵消），噪声分量被抵消（高抵消）
- 如果成立 → 抵消不是浪费，而是噪声过滤机制

**P18（Stage665）**：抵消率的因果干预
- **反事实实验**：强制保留被抵消的分量（将抵消率从56%降到20%），观察：
  - margin如何变化？
  - 其他能力的编码是否受干扰？
  - 模型的生成质量是否下降？
- 实现方法：在特定层注入"反抵消偏置"——将increment_diff_k中的被抵消分量放大

**判伪标准**：
- 如果强制降低抵消率导致margin下降 → 抵消是功能性机制（噪声过滤或精度控制）
- 如果强制降低抵消率不影响margin → 抵消是副产品（架构约束而非功能选择）

#### 任务包C（P19-20）：多能力编码的交叉因果——统一理论的核心验证

**问题诊断**：上次计划的任务包B只完成了消歧的深入分析，其他能力（语法、关系、指代、推理、风格）的编码动力学仍停留在表层。

**核心问题**：消歧上发现的"低维编码+逐层旋转+正交读出"模式，在其他能力上是否存在？

**实验设计**（Stage666-667）：

**P19（Stage666）**：五类能力的全层精确方程验证
- 对语法、关系、指代、推理、风格各3-5个样本
- 验证方程(1)(2)(3)在每类能力上是否成立：
  - 残差流方程误差是否=0？
  - 方向演化方程误差是否=0？
  - 抵消率范围是多少？
  - 关键层带位置在哪？
- 与消歧结果对比

**P20（Stage667）**：跨能力因果干扰实验

- 测量：
  - 注入后当前能力的margin变化（干扰度）
  - 跨能力的正交性是否提供"天然隔离"

**判伪标准**：
- 如果≥3类能力复现"低维+旋转+抵消"模式 → 统一理论有基础

#### 任务包D（P21）：推理能力解码——最大的黑箱

**问题诊断**：推理能力的恢复率<4%（INV-220），是所有能力中唯一无法用低维捕捉的。这可能是"分布式编码"的本质特征，也可能是我们还没找到正确的解码方法。

**假设**：
1. 推理不是单一特征，而是多个子推理的动态组合（如"空间推理"+"逻辑推理"）
2. 推理编码不在hidden state的"方向"上，而在"激活模式"上（即哪些维度被激活，而不只是激活多大）
3. 推理需要时间步的累积效应——单步hidden state无法捕捉

**实验设计**（Stage668）：

**P21（Stage668）**：推理编码的多策略解码
- **策略1：门控模式解码**——不看delta的大小，看delta的符号模式（正/负/零的分布）
- **策略2：子任务分解**——将推理任务分解为子步骤，每步提取独立的编码方向
- **策略3：高维探针**——用1层MLP（而非线性探针）做分类，测试非线性可分性
- **策略4：层级信息整合**——不只看最后一层，而是用所有层的hidden state做联合分类

**判伪标准**：
- 如果任一策略恢复率 > 30% → 推理低维编码假说被确认（只是需要更好的解码方法）
- 如果所有策略恢复率 < 10% → 推理确实是高维分布式编码，需要完全不同的理论框架

**模型**：Qwen3 → DS7B → GLM4

#### 任务包F（P24）：第一性原理候选——从方程体系到"为什么"

**前置条件**：任务包A-E的结果汇总

**目标**：基于P15-P23的所有结果，尝试回答"为什么"——为什么语言编码采用低维+旋转+抵消的结构？有没有更底层的数学约束？

**分析维度**：
1. **信息论约束**：低维编码是否是最小化互信息损失的最优策略？
2. **几何约束**：高维空间中的旋转是否是实现维度分离的"唯一路径"？
3. **能量约束**：抵消率56-78%是否接近某种理论下限（如随机方向的理论抵消率）？
4. **优化约束**：训练过程是否自然地收敛到这种编码结构（通过Grokking式的延迟涌现）？

**产出**：一份"语言编码第一性原理候选"文档，包含：
- 当前最强的理论解释
- 理论的预测（可被后续实验验证/推翻）
- 理论的局限性（哪些现象仍无法解释）

### 六、风险与备用方案

**风险1**：Logit方程修复失败（消歧分量贡献 < 10%）
- **备用**：放弃"消歧方向决定margin"的框架，转向"残差流全局结构决定margin"

**风险2**：多能力统一验证失败（<3类能力复现核心模式）
- **备用**：接受"消歧特例"结论，将理论定位为"多义消歧编码理论"而非"语言编码统一理论"

**风险3**：推理解码全部失败（恢复率 < 10%）
- **备用**：承认推理编码需要完全不同的数学工具（可能是动态系统理论而非线性代数）

### 七、与研究总目标的关系

项目总目标是实现AGI，核心思路是"深度神经网络提取了大脑中的某种数学结构"。

当前进度：我们已经在**消歧能力**上找到了这种数学结构的"影子"——低维编码、逐层旋转、正交读出、抵消过滤。

下一阶段的关键判断：
- **如果多能力统一成功** → 这种数学结构是语言编码的一般规律，接近"语言背后的原理"
- **如果多能力统一失败** → 这种数学结构只是消歧特例，需要回到"拼图"状态重新寻找更底层的模式

无论哪种结果，都将为研究提供明确的下一步方向。
