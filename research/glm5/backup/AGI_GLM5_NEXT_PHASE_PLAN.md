## 0. 当前状态诊断

### 已完成的工作量

- 339+个不变量提出与验证

### 当前理论的核心结构
```
输入文本 → embedding(L0) → ~90度旋转(L0→L1) → 微调(L1→Ln) → unembed → logits
```

### 六个已确认的核心事实
1. **L0对齐≈1.0**：embedding质心方向（高维几何效应，非学习结构）
2. **L0→L1旋转≈90度**：第一层transform的结果，>92%旋转在此完成
3. **旋转平面11维**：跨文本一致（cos=0.76），与语义无关
4. **Logit方程精确**：margin = cos(h,u)·||h||·||u||，误差<0.3%
5. **方向编码为主**：方向承载几乎所有域区分信息（RMSNorm使norm差异仅2.5%）
6. **avg_lp ≈ log(margin) - 1.5**：跨模型稳定

### 当前最大的三个空白
1. **不知道语义编码在哪里**：宏观几何是架构特性，微调层机制不明
2. **不能从文本预测编码**：有精确的描述方程，无预测方程
3. **没有明确的判伪标准**：理论越来越像"总能解释"，缺乏硬性否定条件

## 大任务一：生成化（Predictive）

### 核心问题
> 能不能从文本结构，预测内部方向偏移和读出结果？

### 为什么这是第一优先级
P44-P49证明了一个重要结论：**残差流的宏观几何（方向对齐、旋转角度、旋转平面）完全由架构决定，不由语义决定**。这意味着之前花了大量精力研究的"基底方向"和"旋转"都不承载语义信息。

语义信息一定编码在**旋转后的微调层**中（L1→Ln的<8度角度变化）。但这个微调层的编码机制完全是一个黑箱。

### 三步走路线

#### 步骤1：微调偏移映射（P50-P51）✅ 已完成

**P50结果（Stage696）**：
- **核心确认**：微调偏移方向与语义类别相关（ANOVA F>3）
  - Qwen3：35/36层语义（平均F=5.98）
  - GLM4：39/40层语义（平均F=4.91）
  - DS7B：仅5/28层语义（L2,L3,L21,L22,L23）——语义稀疏
  - Gemma4：12/35层语义（平均F=2.63）
- **DS7B独特性**：语义集中在极少数层（早期L2-L3+晚期L21-L23），中间层不参与语义编码
- **与P48的对比**：P48说"总旋转角度不与语义相关"，P50说"逐层delta方向与语义相关"——互补而非矛盾

**P51结果（Stage697）**：
- 早期层（L0-L2）消融导致margin下降95-99%，top1 token 100%翻转
- 因果性确认：微调偏移确实影响输出
- 方法论限制：简单替换消融对GLM4无效（RMSNorm放大效应）
  - 判定标准：如果ablation后logit margin下降>20% → 该层的Δh_l因果影响输出

#### 步骤2：线性累积验证（P52）✅ 已完成

**P52结果（Stage698）**：
- **INV-354确认**：logits = sum(delta_h_l @ unembed.T) 精确成立（四模型cos>0.995）
- **第一个精确的预测方程**：
  ```
  logits(text) = bias + sum_{l=1}^{n} [(h_l - h_{l-1}) @ U^T]
  ```
  其中U是unembed矩阵。每层向"logit空间"贡献一个向量投票。
- **Embed-only无效**（cos<0.2）——必须经过所有层的累积
- **Margin主要由最后几层贡献**

#### 步骤3：文本→微调偏移预测（P53-P54）✅ 已完成

**P53结果（Stage699）**：
- 替换单token导致delta-h大幅变化（cos_dist 0.07-0.78）
- 敏感层分布因模型而异：
  - Qwen3/GLM4：中前期层最敏感（L8-L21）
  - Gemma4：后期层最敏感（L29-L30）
  - DS7B：分散在各层
- Token替换的logit影响（KL散度）：DS7B和Gemma4最大（>1.0），Qwen3/GLM4较小（<0.35）

**P54结果（Stage700）**：
- **delta-h可被线性近似预测**：overall R2 = 0.84-0.88
- **logit分布可近似**：Pearson r = 0.78-0.90（Qwen3/DS7B通过r>0.8）
- **margin预测失败**：margin r ≈ 0（方向对但精确度不足）
- **per-layer严重过拟合**：train R2 = 0.6-1.0，test R2 < 0
- **Gemma4的100% top1是伪结果**：所有文本预测为同一token `<turn|>`
- GLM4未通过判伪标准（r=0.78 < 0.8，top1=45% < 70%）

**Phase I结论**：生成化路线"方向正确但精度不足"——text→delta→logit链在原理上可行（P52线性累积精确），但delta-h预测需要更好的文本特征提取或非线性模型。

### 判伪标准（已更新）
- ~~步骤1~~：✅ 已确认——微调偏移确实与语义相关（ANOVA F>3），无判伪触发
- ~~步骤3~~：⚠️ 部分通过——Qwen3/DS7B logit Pearson r>0.8，但margin r≈0且per-layer test R2<0
- **关键发现**：线性预测delta-h的瓶颈不在线性累积（P52已证明精确），而在"文本特征→delta-h"的映射
- **生成化路线判定**：方向正确但精度不足，需要Phase II（判伪化）加固基础理论，再决定是否需要非线性模型

## 大任务二：判伪化（Falsifiable）

### 核心问题
> 在什么条件下，当前理论v4.1必须被推翻或重写？

### 为什么这是第二优先级
当前理论已经积累了大量"支持证据"，但缺乏"否定条件"。一个没有判伪标准的理论不是科学理论——它只是在收集现象，而不是在解释规律。

### 需要判伪的五个核心命题

#### 命题1：语义编码在方向空间中
- **当前证据**：P36确认方向>norm承载域区分信息
- **判伪条件**：如果存在某种操作（如仅改变norm、仅改变attention pattern），在保持方向不变的情况下，能显著改变输出语义 → 命题1被推翻
- **实验设计**：P55 Norm-only因果实验
  - 固定h的方向，仅修改||h||，测量logit变化
  - 如果logit top-1准确率下降<5% → norm不重要（命题1确认）
  - 如果下降>20% → norm承载语义（命题1推翻）
- **P55结果**：✅ 命题1确认——四模型norm缩放到0.1x-20x时，top-1完全不变(8/8)，logit r>0.996。方向完全决定输出，norm几乎不影响语义选择。

#### 命题2：旋转是架构特性（非语义操作）
- **当前证据**：P48确认旋转角度与类别/长度无关
- **判伪条件**：如果通过因果干预（如在特定层注入特定方向的bias）能改变旋转平面，且该改变导致语义处理能力下降 → 旋转有语义功能 → 命题2被推翻
- **实验设计**：P56 旋转平面干预实验
  - 在L0→L1之间注入一个旋转矩阵，将旋转平面从11维扩展到100维
  - 如果模型能力（困惑度）下降>10% → 旋转平面大小有语义功能
  - 如果不变 → 旋转确实是架构副作用
- **P56结果**：✅ 命题2确认——仅用K=1维PCA投影就能保持r>0.9且top-1=100%（Qwen3/DS7B/GLM4），说明有效维度极低（<<11），与"11维旋转平面"一致。

#### 命题3：11维旋转平面是Transformer通用特征
- **当前证据**：四模型PCA90=11一致
- **判伪条件**：在非Transformer架构（如Mamba、RWKV）上发现PCA90≠11 → 命题3被推翻
- **实验设计**：P57 跨架构旋转分析
  - 在Mamba-1.4B和RWKV-6-7B上重复P47实验
  - 如果PCA90≠11±2 → 旋转维度是架构特定的
- **P57结果**：⏳ 未执行（需要Mamba/RWKV本地模型）

#### 命题4：avg_lp ≈ log(margin) - 1.5 是普适方程
- **当前证据**：四模型上验证
- **判伪条件**：在明显不同的任务上（如代码生成、数学推理、多语言翻译），常数C≠1.5±0.3 → 命题4被推翻
- **实验设计**：P58 任务依赖性验证
  - 在纯代码文本、纯数学文本、纯中文文本上分别测量C值
  - 如果C值跨任务差异>1.0 → 方程不是普适的
- **P58结果**：❌ 命题4被推翻——C值跨任务差异巨大：
  - Qwen3: C范围1.3-3.7 (range=2.4)
  - DS7B: C范围1.1-5.3 (range=4.2)
  - GLM4: C范围1.8-6.6 (range=4.9)
  - Gemma4: C范围10.0-15.2 (range=5.2)
  - 规律：代码/数学C较低(1-3)，通用英文C较高(4-6)，中文C最高(3-14)
  - **C不是常数，而是任务依赖的变量**。原方程需修改为 avg_lp ≈ log(margin) - C(task)

#### 命题5：DS7B的独特性在纠缠度（std=0.206）
- **当前证据**：DS7B纠缠度远高于其他模型（0.065-0.125）
- **判伪条件**：如果能在非RL模型中通过微调复制DS7B的纠缠度，且该微调不改变模型能力 → 纠缠度不是RL的独特结果
- **实验设计**：P59 纠缠度复制实验
  - 用LoRA在GLM4上训练，目标是最小化纠缠度std到0.206
  - 如果可以轻易复制 → 纠缠度不是RL独有
  - 如果不能（或导致能力崩溃）→ 纠缠度确实是RL的关键产物

### 判伪矩阵总览

| 命题 | 核心主张 | 推翻条件 | 实验 | 结果 |
|------|---------|---------|------|------|
| P1 | 语义在方向空间 | norm-only操作改变语义>20% | P55 | ✅ 确认：norm缩放不影响top1 |
| P2 | 旋转=架构特性 | 扩展旋转平面导致能力下降>10% | P56 | ✅ 确认：K=1维即可保持top1 |
| P3 | 11维旋转=通用 | 非Transformer PCA90≠11±2 | P57 | ⏳ 未执行 |
| P4 | C=1.5是普适的 | 跨任务C差异>1.0 | P58 | ❌ 推翻：C范围1.3-15.2 |
| P5 | DS7B独特=纠缠度 | 非RL模型可复制纠缠度 | P59 | ⏳ 未执行 |

**关键结论**：5个命题中，3个已验证（P1/P2确认，P4推翻），2个待执行（P57/P59）。P4的推翻要求理论修改：avg_lp方程不是普适的，C是任务依赖的。

**关键原则**：以上任何一个命题被推翻，都要求对v4.1进行**结构性修改**（不是修补，而是重写受影响的部分）。

## 大任务三：基元化（Primitives）

### 核心问题
> 最小编码基元到底是什么？

### 为什么这是第三优先级（但最终最重要）
前两个任务（生成化和判伪化）都在现有框架内工作。但如果编码基元本身被误解了（比如编码不是"方向偏移"而是"注意力模式的拓扑结构"），那前两个任务的根基就有问题。

基元化是通往第一性原理的必经之路。

### 五种候选基元

#### 候选A：单一方向偏移
- **假设**：语义通过hidden state在特定方向上的投影值编码
- **当前支持**：P36（方向承载域区分）、Logit方程（cos(h,u)项）
- **问题**：P48证明宏观方向偏移与语义无关。微观方向偏移（<8度）是否足够？

#### 候选B：低维子空间
- **假设**：语义通过hidden state在特定低维子空间中的坐标编码
- **当前支持**：P47（11维旋转平面）、P35（PCA90=13-45维）
- **问题**：11维旋转平面已被证明与语义无关。是否存在另一个"语义子空间"？

#### 候选C：层间轨迹
- **假设**：语义通过hidden state跨层的轨迹形状编码（不仅仅是起点和终点）
- **当前支持**：P38层动力学（三种模式）
- **问题**：如果轨迹重要，那么中间层ablation应该比末层ablation影响更大

#### 候选D：注意力模式
- **假设**：语义编码不在hidden state的值中，而在注意力权重矩阵的模式中
- **当前支持**：P41（DS7B注意力-方向强耦合r=0.528）
- **问题**：注意力模式是编码的"原因"还是"结果"？

#### 候选E：门控/调制模式
- **假设**：语义通过FFN的门控开关组合编码（某些神经元开/关表示不同语义）
- **当前支持**：Phase5的"旋转器"分析
- **问题**：门控模式是否比连续方向偏移更能解释语义区分？

### 基元判定实验（P60-P64）

**P60结果（Stage703）：**
- **E_gate（门控幅度）破坏最大**：top-1=0%, Pearson r=-0.04 to 0.14, KL=18-21
- **B_subspace（子空间投影）次之**：top-1=0%, r=0.08-0.30, KL=7-8
- **A_direction（方向偏移）中等**：top-1=20-60%, r=0.86-0.94, KL=0.5-2.2
- **C_trajectory（层间轨迹）无影响**：top-1=100%, r=1.0, KL=0
- **D_attention（注意力模式）无影响**：top-1=100%, r=1.0, KL=0
- **关键洞察**：轨迹和注意力消融不影响输出，因为P52证明logits=sum(delta_h@U.T)仅依赖最终h，不依赖路径。这说明C/D不是编码基元而是中间过程。
- **真正重要的是**：(1)h所在的子空间（B），(2)每层delta-h的大小/幅度（E_gate），(3)h的方向（A_direction）

**P61结果（Stage704）：**
- **A方向容量**：~100 bits（四模型effective rank=9，95%方差8-9维）
- **B子空间容量**：~70 bits（95%方差6-8维）
- **E门控容量**：327-852 bits（取决于sparsity：DS7B=9.1%最稀疏，Qwen3=25.7%最稀疏）
- **C轨迹容量**：23,000-125,000 bits（远超需要）
- **D注意力容量**：69,000-147,000 bits（远超需要）

**P63结果（Stage704）：**
- **仅需1%维度即可保持100% top-1**（Qwen3: 25/2560, DS7B: 35/3584, GLM4: 40/4096, Gemma4: 15/1536）
- **Delta-h稀疏性**：5%阈值保留40-47%维度，10%保留15-26%，20%保留5-10%
- **Per-layer delta-h PCA**：早期/晚期层delta较稀疏（d90=20-325），中期层较密（d90=1071-1820）

**P62结果（Stage705）**：
- **A+E无正向协同**：d_AE <= max(d_A, d_E)，门控幅度E单独主导
- 任何含E的组合damage ≈ 3.0（破坏严重），不含E的最大damage ≈ 2.6
- E_only: margin ratio = 4.4-30.2x（异常增大），说明等化后margin反而增加但不准确
- A_only: margin ratio = 0.83-1.02x（几乎不变），方向偏移仅轻微影响margin
- 结论：**编码基元不是独立的组合，门控幅度是主导因素**

**Unembed Rank分析（Stage705）**：
- 四模型unembed均为满秩或接近满秩：Qwen3=2560, DS7B=3134, GLM4=3935, Gemma4=1536
- hidden state effective rank=9是模型内部的涌现属性，不是unembed的限制
- Top-1 SV远大于其余：SV[1]/SV[0] = 0.20-0.35

**P55/P60矛盾解决（Stage705）**：
- P55：均匀缩放h_final → softmax(ax) = softmax(x) → argmax不变
- E_gate：改变各层delta-h的相对幅度 → h_final方向改变
- delta-h的Max/Min ratio = 84-198x，CV = 1.5-2.0
- 关键：语义编码在**层间贡献的模式(pattern)**中，不在总幅度中

**Margin分析（Stage705）**：
- 1%维度保持margin ratio = 0.93-1.09（几乎完美）
- 低维PCA投影不仅保持top-1还精确保持margin

**P64结果（Stage706）：**
- **rank=9被推翻！** 用30条文本时rank=31（满秩！），用15条文本时rank=15。之前的"rank=9"是10条文本的样本数上限——effective rank被样本数限制。
- **h_final PCA与unembed PCA几乎正交**：四模型alignment=−0.015到0.020（接近0），语义编码方向与unembed矩阵方向无关——是模型内部涌现的独立结构。
- **随机投影保持rank**：投影到32D-1024D后rank不变(15)——符合Johnson-Lindenstrauss引理，高维空间的距离关系在低维中被保持。
- **scale不变性**：0.1x-10x缩放后rank完全不变——确认rank是纯粹的几何性质。
- **9个主成分有明确语义**（Qwen3）：
  - PC0(43.5%)：语言类型（英文vs中文，gen-cn差异=135.6）
  - PC1(14.7%)：文本格式（代码标识符/中文词汇）
  - PC2(6.2%)：逻辑类型（True/FALSE/RNA/Carbon）
  - PC3(3.9%)：生物学（plants/sunlight/organisms）
  - PC5(2.9%)：ML术语（layers/Gradient/neural）
- **9-PC重建精度**：logit cos=0.967-0.981，top-1=60%——方向高度保真但精确排序有损

**P65结果（Stage707）：**
- **100文本有效维度**：Qwen3/GLM4/Gemma4=100(满秩)，DS7B=89(有内部压缩！)
- **每类20文本**：全部rank=20(满秩)
- **方差解释**：Top-5=82-90%，Top-10=86-94%，Top-20=90-96%，Top-50=96-99%
- **SV衰减**：SV[1]/SV[0]=0.34-0.57, SV[5]/SV[0]=0.10-0.14
- **DS7B独特性再次确认**：唯一在100文本下不满秩的模型，信息确实被压缩到<100维
- **结论**：真实有效维度远高于之前的"9"，至少>100（三模型）或约89（DS7B）

**P66/P67**：P66(Logger bug修复待做)，P67(跨模型维度不匹配待修复)

### 判伪标准
- 如果P60发现**所有五种基元的消融效果相似**（下降都在10-20%范围内）→ 编码基元不是单一的，而是这些机制的复杂组合
- 如果P61发现**所有基元的信息量都远超需要**（比如只需100 bits就能区分所有语义类别）→ 说明存在大量冗余，真正的编码可能更简洁
- 如果P63发现**需要>80%的维度才能保持准确率**→ 编码是高维分布式表示，不存在简单的"基元"

## 执行优先级与时间规划

### 阶段划分

```
Phase I（生成化）：P50-P54
  目标：建立 文本→微调偏移→logit 的预测链
  预计：3-5个stage（P50-P54）
  核心产出：第一个能"事前预测"的方程
  判停条件：如果预测准确率<30%，停止并转入基元化

Phase II（判伪化）：P55-P59
  目标：为v4.1的5个核心命题设定推翻条件
  预计：5个stage（P55-P59）
  核心产出：v5.0（如果命题被推翻）或v4.2（如果全部确认）
  判停条件：如果>2个命题被推翻，需要全面重写理论

Phase III（基元化）：P60-P64
  目标：确定最小编码基元
  预计：5个stage（P60-P64）
  核心产出：编码基元的数学定义 + 第一性原理推导
  判停条件：如果无法找到单一基元，需要重新考虑组合基元理论
```

### 三个阶段的依赖关系
```
Phase I ──→ 如果预测失败 ──→ Phase III（先确定基元）
   │                              ↑
   └──→ 如果预测成功 ──→ Phase II ─┘
```

即：
- 如果生成化成功 → 说明当前框架基本正确，用判伪化加固
- 如果生成化失败 → 说明当前框架缺少关键组件，需要先做基元化

### 每个阶段的"硬停"标准
- Phase I硬停：P50发现没有任何层与语义相关 → 需要重新思考语义编码位置
- Phase II硬停：>2个命题被推翻 → 需要全面重写理论
- Phase III硬停：无法找到<总维度20%的最小充分基元 → 编码可能是真正的全维分布式，需要全新理论框架

## 一句话总结

> P44-P49告诉我们：残差流的"主干传输规律"（旋转90度、11维平面）是架构的数学副产物，不是语言的本质。真正的语言编码发生在主干之外的"微调信号"中。下一阶段的使命不是继续测绘主干，而是找到微调信号的编码基元，建立预测方程，设定判伪标准——从"知道它长什么样"推进到"知道它为什么必然长这样"。
