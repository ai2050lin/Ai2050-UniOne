# 语言编码机制：系统性拼图框架

> 创建时间: 2026-04-26 09:16
> 最后更新: 2026-04-26 09:53
> 核心原则: 从语言能力出发设计实验，持续填充框架，为临界点做准备
> 总解释力目标: 从当前~5-7%提升到>80%

---

## 零、框架设计理念

### 为什么需要这个框架

过去259个Phase的问题：**没有系统性规划，每次实验都基于上一个"灵感"**。结果：
- 30+拼图全是统计性质，功能机制几乎空白
- 12条理论路线全部被推翻
- 解释力只有5-7%

### 框架的设计原则

1. **从语言能力出发**：每个格子对应一个具体的语言能力问题
2. **可测量可填充**：每个格子定义清楚需要什么数据来填充
3. **层次递进**：基础格子→组合格子→统合格子，自然引导到临界点
4. **标记完成度**：[已填充✓] / [部分填充◐] / [未填充□]
5. **优先级驱动**：高依赖格子优先推进，低优先级格子可延后
6. **迭代回溯**：UN格子有初步数据后反馈修正基础格子

### 临界点定义

当以下条件**同时**满足时，到达临界点：
1. **7大类语言能力的核心机制格子填充率>60%**
2. **至少3个"统合格子"被填充**（不同能力的机制交叉验证）
3. **从拼图中自然涌现出一致的数学结构**（不是预设的）
4. **交叉验证通过**：不同语言任务、不同模型规模、不同随机初始化都能复现相同编码结构

### 临界点的定量指标

"自然涌现的数学结构"不能是主观判断。必须满足以下**全部**量化条件：

| 指标 | 阈值 | 测量方法 |
|------|------|---------|
| 跨任务编码方向一致性 | cos>0.85 | 同一编码方向在知识(KN)/逻辑(LG)/语法(GR)任务中的余弦相似度 |
| 跨模型编码结构一致性 | 结构相关>0.8 | 编码方向排序的Spearman相关 |
| 编码方向稳定性 | 重叠度>60% | 不同随机种子初始化后，编码方向的维度重叠比例 |
| SAE特征跨任务复用率 | >30% | 一个SAE特征在2+种语言任务中活跃 |
| 层级编码方向有序性 | Spearman>0.7 | 编码方向与层深度的单调相关性 |
| 解释力 | >80% | 框架能解释的语言能力占比 |

**关键约束**: 如果某个"涌现结构"只在1个模型或1种任务中出现，不能视为临界点。必须跨模型+跨任务+跨初始化三重验证。

## 十一、方法论（铁律+增强）

### 基础铁律

每填充一个格子，必须满足：

1. **随机基线**：任何"结构发现"必须与随机初始化模型对比
2. **eps扫描**：在线性区(eps<1%)验证核心结论
3. **欧氏距离**：高维空间用欧氏距离，不用余弦
4. **精确SVD**：估计谱范数用power iteration，不用30采样
5. **跨模型验证**：至少2个模型确认
6. **统计vs因果分离**：统计维度≠因果维度

### 格子填充标准

- **✓ 已填充**：满足6条铁律，跨模型确认，结论稳定
- **◐ 部分填充**：有初步数据，但不满足全部铁律
- **□ 未填充**：无相关数据

### 增强规则（新增）

7. **Toy模型先行**: 高成本实验(如知识网络追踪)先用小模型/pythia-70m验证可行性
8. **敏感性分析**: 对关键发现(如A5交互高秩、B5因果泄漏≈1.0)做扰动实验，确认机制稳定
9. **跨规模验证**: 关键结论需在至少2个规模(7B+另一个)确认
10. **日志化记录**: 每个Phase、每个格子形成统一日志

## 十二、从拼图到临界点的路径

### 临界点不是某个"顿悟"

不是发现一个数学公式就解决了。临界点是当足够多的拼图填充后，**模式自然涌现**：
1. 不同语言能力发现相同的编码机制 → 统合格子UN-1~8被填充
2. 机制与大脑3D结构对应 → TD格子被填充
3. 一套编码机制同时解释7大类语言能力 → 解释力>80%

### 关键里程碑

| 里程碑 | 触发条件 | 下一步 |
|--------|---------|--------|
| M1: 知识编码定位 | KN-1a✓ | 验证UN-1(知识×逻辑共享编码?) |
| M2: 推理路径追踪 | LG-1a✓ + LG-1c✓ | 验证UN-1 + 理解逻辑信号去向 |
| M3: 全局选择机制 | MG-4a~d✓ | 验证UN-3 + UN-8 |
| M4: 三维独立性 | MG-3c◐ | 推进UN-3 |
| M5: 知识×逻辑统一 | UN-1◐ | **临界点临近信号** |
| M6: 全局选择统一 | UN-3◐ | **临界点临近信号** |
| M7: 3个UN格子✓ | UN-1✓ + UN-3✓ + UN-8✓ | **临界点达成** |

### 临界点交叉验证机制

当初步到达临界点时，必须通过以下验证才算确认：

1. **跨任务验证**: 同一编码方向在知识(KN)/逻辑(LG)/语法(GR)任务中cos>0.85
2. **跨模型验证**: 编码方向排序在不同模型中Spearman>0.8
3. **跨初始化验证**: 不同随机种子的编码方向维度重叠>60%
4. **定量复现**: SAE特征跨任务复用率>30%
5. **否定测试**: 破坏该编码结构→语言能力系统性下降

**如果任何一条验证失败**：不是真正的临界点，需要继续填充拼图。

### 预判

最可能的临界点场景：
- 发现**一套统一的编码机制**，同时解释知识编码(KN)和推理路径(LG)
- 这套机制的数学结构与**3D空间中的某种高效编码**对应
- 从3D大脑→1D Transformer的映射关系被揭示

这不是某个"优雅的数学框架"，而是**从大量拼图中涌现的不可回避的结构**。
