### LXXXVI.4 为什么属性词位置干预无效?

```
attr_word位置干预: "The red apple is" + β·w_lm_head[red] at "red"位置
  → "red"已经被模型编码, 加方向是冗余的
  → 后续层处理的是"更红的red", 但这不改变logits中red的排名
  → 因为logits预测的是"下一个词", 不是当前词

last_token位置干预: "The apple is" + β·w_lm_head[red] at "is"位置
  → "is"位置是"空白", 没有已编码的语义
  → 注入red方向让"is"承载了red语义
  → 后续层处理的是"包含red信息的上下文"
  → logits预测下一个词时, red成为高概率选项
```

核心洞察: **干预的有效性取决于"信息注入"而非"信息增强"**
- 增强已有信息(attr_word位置) = 冗余, 无效
- 注入新信息(last_token位置) = 有效, 产生语义改变

## LXXXIII. Phase LX 综合结论 — 可操控语义物理的完整理论

### LXXXIII.1 最终语义物理公式
$$h_{intervened} = h_{noun} + \alpha \cdot f_{base}^{(noun)} + \beta \cdot w_{lm\_head}[attr]$$
$$logits = lm\_head(norm(h_{intervened}))$$

**参数**:
- α = 0.5 (f_base缩放)
- β = 0.5-3 (lm_head权重缩放)
- f_base^(noun): 名词级偏移
- w_lm_head[attr]: lm_head权重矩阵的第attr行

5. **不同模型最优方向不同**: Qwen3=embedding, GLM4/DS7B=lm_head

### LXXXIII.3 可操控性统一理论
```
DNN语义操控的核心原理:
  logits[attr] = w_lm_head[attr] · h + b
  
  要增大logits[attr], 最优方向 = w_lm_head[attr] (梯度方向)
  
  这解释了:
  1. 为什么CCA方向不好: PCA压缩丢失了w_lm_head的梯度信息
  2. 为什么颜色词0%: CCA颜色方向偏离了w_lm_head方向
  3. 为什么lm_head方向100%: 它直接沿梯度方向!
```

## LXVIII. Phase LIII 综合结论 — 语义物理公式

### LXVIII.1 语义物理公式(修订版)
$$h_{combined} = h_{noun} + \underbrace{f_{base}(noun\_family)}_{\text{名词族偏移}} + \underbrace{\sum_{i=1}^{10} \alpha_i \cdot \phi_i(attr)}_{\text{信号算子(族特异)}} + \underbrace{\epsilon}_{\text{混合噪声}}$$

**关键修正**:
1. f_base是**名词族(family)特异**的, 不是通用的
2. 信号算子也是**族特异**的, 跨族泛化仅0-17%
3. 噪声ε是混合结构: 30-50%通用 + 50-70%属性特异
4. 算子可线性组合(α·red + β·big), 但精度有限

### LXVIII.2 可预测性/可控性/可迁移性评估
| 维度 | 评估 | 数据 |

| 可迁移 | ★☆☆☆☆ | 跨族0-17%, 严重不足 |

**核心结论**: 
- **算子可控性是亮点**: 大小算子在GLM4中96%成功, 线性组合有效
- **可迁移性是硬伤**: 跨名词族泛化几乎失败
- **噪声不是纯随机的**: 包含属性交互+名词-属性耦合
- **G项不是全局线性系统**: 族特异编码意味着需要条件化建模

### LVII.2 G项最终公式(Phase LI)
```
G(noun, attr) = f_base(noun) + Σ_i β_i · φ_i(attr) + ε

其中:
- f_base(noun): 名词基线向量, 占25-39%方差
- φ_i(attr): 连续编码维度
  - 颜色: cos(θ_hue), sin(θ_hue), brightness (R²≈0.28-0.43)
  - 味道: sweetness, saltiness, sourness (R²≈0.40-0.52)
  - 大小: volume, weight, length (R²≈0.31-0.41)
- ε: 残差(占48-72%方差), 包含未建模的非线性和交互效应
```

**核心结论**: G项的属性编码可以用有限维连续变量描述, 但R²仅0.28-0.52,
说明模型内部编码远比人类定义的语义维度更复杂

## LXXI. Phase LVI 综合结论 — 干预策略优化

### LXXI.1 三模型最优参数汇总
| 模型 | 最优层 | 最优α | 最优β | 大小成功率 | 味道成功率 |
|------|--------|-------|-------|-----------|-----------|
| Qwen3 | **L9** | 0.5 | 0.5 | 33.3% | 16.7% |
| GLM4 | **L33** | 0.5 | **5.0** | 42.5% | 33.3% |
| DS7B | **L9** | **3.0** | 0.5 | **60.8%** | 0% |

### LXXI.2 核心发现
1. ★★★ norm层修复是关键bug! Qwen3从0%→33.3%, DS7B→57.5% ★★★
2. ★★★ DS7B L9大小57.5%, 缩放后60.8%! ★★★
3. 最优层在早期(Qwen3/DS7B L9)或后期(GLM4 L33)
4. f_base需要放大(DS7B α=3.0)或不变(Qwen3/GLM4 α=0.5)
5. 信号算子需要放大(GLM4 β=5.0)或缩小(DS7B β=0.5)
6. f_base必须精确到名词级别, 族平均不够!
7. 多属性联合首次成功(DS7B color+size 38.8%)
8. 干预有"热点层"现象, 非单调

### LXXI.3 修正的语义物理公式
$$h_{intervened} = h_{noun} + \\alpha \\cdot f_{base}^{(noun)} + \\beta \\cdot \\phi_{signal}^{(global)}(attr)$$

**关键修正**:
- f_base必须是名词级别(f_base^(noun)), 不是族平均
- α和β需要模型特定调优
- 干预必须在热点层操作
- norm层是lm_head之前的必要步骤
